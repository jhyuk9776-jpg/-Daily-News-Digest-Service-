# AI 공통 작업 맥락

작성일: 2026-06-29
프로젝트: AI 뉴스 다이제스트

이 파일은 Codex, Claude, ChatGPT 등 어떤 AI가 작업하더라도 같은 방향으로 판단하도록 만드는 공통 기준 문서다. 새 AI가 이 프로젝트에 참여하면 먼저 이 파일을 읽고, 세부 내용은 `기획/` 문서로 확인한다.

## 1. 프로젝트 한 줄 정의

매일 아침 신뢰할 수 있는 뉴스 출처에서 주요 사건을 수집하고, 중복을 줄인 뒤, AI가 판단과 해석을 덜어낸 사실 중심 요약으로 정리하는 데일리 뉴스 다이제스트 시스템.

## 2. 장기 비전

장기적으로는 단순 뉴스 요약기가 아니라 AI가 편집장 역할을 하는 뉴스 브리핑 플랫폼을 목표로 한다.

장기 확장 방향:

- 이슈별 기사 그룹화
- 중요도 평가
- 개인 관심사 기반 필터링
- AI 해설과 배경 설명
- 웹/PWA와 모바일 앱
- 이메일, 노션, 푸시 알림
- 후속 보도 추적
- 뉴스 기반 문제/기회 신호 추출

단, 이 장기 비전은 지금 당장 만들 범위가 아니다. 현재는 Phase 1의 작은 결과물을 먼저 완성한다.

## 3. 현재 Phase 1 목표

Phase 1은 `판단 없는 사실 중심 뉴스 다이제스트`를 매일 생성하는 최소 시스템을 만드는 것이다.

현재 목표:

- 고정 뉴스 출처에서 기사 수집
- 날짜와 분야 기준으로 선별
- 중복 기사 줄이기
- AI로 1~3문장 사실 중심 요약
- 원문 링크 포함
- 날짜별 마크다운 파일 저장

예상 결과물:

```text
News/YYYY-MM-DD.md
```

## 4. Phase 1에서 하지 않는 것

아래 기능은 좋은 아이디어여도 Phase 1에서는 넣지 않는다.

- 웹/앱 UI
- 개인화 추천
- 푸시 알림
- 의견, 논평, 전망 생성
- AI 해설 또는 시장 영향 분석
- 커뮤니티 반응 분석
- 유료 뉴스 전문 스크래핑
- 대규모 DB 설계
- 복잡한 서버 아키텍처

필요하면 Phase 2 후보로 기록하되, Phase 1 구현을 흔들지 않는다.

## 5. 제품 원칙

1. 사용자는 뉴스를 많이 읽고 싶은 사람이 아니라, 세상의 흐름을 효율적으로 파악하고 싶은 사람이다.
2. 요약은 판단을 대신 내려주지 않는다. 판단할 재료를 정리한다.
3. 출처 링크는 반드시 남긴다.
4. 요약 품질보다 먼저 중요한 것은 출처, 날짜, 중복 제거의 신뢰성이다.
5. 작은 결과물을 매일 쓸 수 있게 만든 뒤 확장한다.

## 6. 요약 원칙

AI 요약은 아래 규칙을 지킨다.

- 사건, 수치, 날짜, 발언 주체를 우선한다.
- 평가어와 감정적 표현을 줄인다.
- 원문에 없는 배경 지식을 덧붙이지 않는다.
- 전망과 추측은 원문에 명시된 경우에만 `누가 말했다` 형태로 적는다.
- 확인되지 않은 내용은 요약하지 않는다.
- 한 항목은 짧게 유지한다.

피해야 할 표현:

- 논란이 커지고 있다
- 충격을 주고 있다
- 큰 파장이 예상된다
- 업계가 주목하고 있다

위 표현은 원문에 있더라도 가능하면 더 구체적인 사실로 바꾼다.

## 7. 현재 추천 결정안

아직 사용자의 최종 승인이 필요한 항목은 `추천`으로 둔다.

| 항목 | 현재 방향 |
|---|---|
| 분야 | 경제, 사회, 세계, IT/테크 추천 |
| 정치 | Phase 1에서는 제외 추천 |
| 기사 수 | 분야별 2건, 하루 총 8건 추천 |
| 결과물 | `News/YYYY-MM-DD.md` 추천 |
| 구현 방식 | 로컬 Python 스크립트 |
| 실행 방식 | 처음은 수동 실행, 이후 자동화 |
| 자동화 후보 | GitHub Actions 또는 macOS launchd |
| AI 모델 | 저비용 모델로 시작 후 품질 비교 |
| 첫 달 비용 상한 | 5~15달러 추천 |

## 8. 작업 우선순위

작업할 때는 아래 순서를 우선한다.

1. 기획과 결정 로그의 미정 항목을 줄인다.
2. 수집, 선별, 요약, 전달을 독립 부품으로 만든다.
3. 먼저 수동 실행으로 결과물 품질을 확인한다.
4. 그다음 자동화를 붙인다.
5. 웹/앱은 데이터 파이프라인이 안정화된 뒤 검토한다.

## 9. 문서 읽는 순서

새 AI가 프로젝트 맥락을 잡을 때는 다음 순서로 읽는다.

1. `AI_CONTEXT.md`
2. `기획/README.md`
3. `기획/01-product-brief.md`
4. `기획/02-mvp-spec.md`
5. `기획/03-system-architecture.md`
6. `기획/04-decision-log.md`
7. `기획/05-roadmap.md`
8. `기획/회의내용정리/01-today-plan-and-cost.md`

## 10. AI 협업 규칙

AI는 이 프로젝트에서 다음 태도로 작업한다.

- 큰 비전을 말할 수는 있지만, 현재 구현 범위를 함부로 키우지 않는다.
- 결정된 내용과 추천 상태를 구분한다.
- 문서와 코드가 충돌하면 먼저 사용자에게 알려 정리한다.
- 새 아이디어는 바로 구현하지 말고 Phase 2 후보 또는 결정 로그에 남긴다.
- 사용자가 구현을 요청하면 작게 만들고 실제 실행 가능한 형태로 검증한다.
- 비용이 늘어나는 선택은 먼저 비용 가정을 밝힌다.

## 11. 현재 핵심 질문

다음 질문이 확정되면 Phase 1 구현에 바로 들어갈 수 있다.

1. 정치 분야를 Phase 1에서 제외할 것인가?
2. 해외 뉴스는 세계/IT 분야에 포함할 것인가?
3. 하루 8건이 적당한가, 10~12건으로 늘릴 것인가?
4. AI 모델은 OpenAI 우선으로 갈 것인가?
5. 자동화는 GitHub Actions를 1순위로 둘 것인가?
6. 뉴스 출처 목록과 출처 우선순위는 어떻게 둘 것인가?
