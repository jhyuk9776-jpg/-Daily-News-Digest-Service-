# 감점 사전 설계 프롬프트 (객관성 점수 · observe-only)

> 이 파일은 **AI에게 1회 보내** 우리 시스템의 감점 사전/룰을 확장·설계받기 위한 프롬프트다.
> 배경: `기획/일정/일정-로드맵.md §9`(매체 객관성 동적 점수, 구 05-roadmap §6), `AI_CONTEXT.md §6`(피해야 할 표현),
> `src/objectivity.py`(현재 채점기). 받은 결과를 사람이 검수·취사선택해 코드에 싣는다.
> 채점 코드(등급 가중치 반영 + 당일 감점 메모 집계)는 이 사전 확정 **다음 단계**에서 수정한다.

# 역할
너는 **한국어 뉴스 객관성 채점 사전 설계자**다. 우리는 매일 아침 사실 중심 뉴스 다이제스트를
만들고, 그 전에 수집한 기사(하루 ~1000건)의 **제목 + RSS 요약**을 채점해 매체별 객관성 평판을
누적한다(관찰 전용, 랭킹 미반영).

# 반드시 지킬 제약 (중요)
1. **런타임에 AI 판단이 없다.** 채점은 파이썬이 **결정적 문자열/정규식 매칭**만 수행한다.
   따라서 네 출력은 반드시 **코드가 그대로 매칭할 수 있는 문자열 또는 파이썬 `re` 정규식**이어야 한다.
   "감정적으로 보이면 감점" 같은 판단 규칙은 쓸 수 없다. 표현을 구체적 문자열/패턴으로 특정하라.
2. **정밀도 우선(재현율보다).** 이 사전은 사실 보도를 잘못 깎으면 안 된다. 애매하면 넣지 마라.
   조금 놓치는 것보다 사실 기사를 오감점하는 것이 훨씬 나쁘다.
3. **단일 모호어 금지.** "충격", "참사", "비상", "경악" 같은 **한 단어 단독**은 지진·사고 등
   사실 문맥에도 흔하므로 감점어로 넣지 마라. 감점은 **평가·조롱·전망·낚시가 분명히 드러나는
   다어절 표현/구문**만 대상으로 한다. 단독 모호어는 `exclusions`에 이유와 함께 적어라.
4. **한국어 뉴스 제목체**를 전제로 하라(체언 종결, 생략, 물음표 낚시 등).

# 채점 메커니즘 (네 출력이 어떻게 쓰이는지)
- 모든 기사는 100점에서 시작(BASELINE=100), 최저 0점(FLOOR=0).
- 매칭된 표현마다 그 표현의 `weight`만큼 감점한다(한 기사에 여러 개면 합산, 같은 표현 반복 등장 시 횟수만큼).
- 등급별 기본 가중치(가이드, 필요하면 항목별로 조정 제안 가능):
  - `strong` = 15 : 조롱·단정·낚시로 사실을 왜곡/과장하는 표현
  - `medium` = 8 : 평가·전망·감정을 드러내는 표현
  - `weak` = 3 : 약한 과장·주관적 묘사

# 우리의 객관성 원칙 (AI_CONTEXT §6)
요약·선별은 아래를 지킨다:
- 사건, 수치, 날짜, 발언 주체를 우선한다.
- 평가어와 감정적 표현을 줄인다.
- 원문에 없는 배경/추측을 덧붙이지 않는다.
- 전망·추측은 "누가 말했다" 형태로 명시된 경우만 인정한다.

피해야 할 표현(감점 사전의 시드 — 여기서 출발해 **대폭 확장**하라):
- "논란이 커지고 있다"
- "충격을 주고 있다"
- "큰 파장이 예상된다"
- "업계가 주목하고 있다"

# 앵커 예시 (판단 기준 보정용)
- **감점 대상(조롱조 단정):** "슬금슬금 오르더니 이젠 4%, 반도체가 다 했네…성장률 전망 일제히 상향"
  → "반도체가 다 했네"는 성과를 조롱조로 단정하는 평가 표현. `~가 다 했네` 패턴으로 잡아라.
- **만점 유지(사실 중심):** "6월 외환보유액 3억7000만 달러↑…금융기관 외화예수금 증가 영향"
  → 수치·주체·사실만. 감점 없음.

# 작업
1. 위 원칙·시드·표본에서 **감점 대상 표현**을 최대한 찾아낸다(제목체의 낚시·조롱·전망·감정 포함).
2. 각 표현을 `strong / medium / weak` 등급으로 분류하고 가중치를 정한다.
3. 표현이 형태 변형되는 경우(어미·조사 변화)는 `phrase`(부분일치) 대신 `regex`(파이썬 `re`)로 일반화한다.
   단, 정규식은 과탐하지 않게 좁게 작성한다.
4. 사실 문맥에 자주 나와 감점하면 안 되는 단어는 `exclusions`에 이유와 함께 적는다.
5. 각 항목에 **한 줄 한글 근거 라벨**을 붙인다. 이 라벨은 매일 "감점 요약 메모"에 그대로 노출되므로
   짧고 사람이 읽기 좋게(예: "조롱조 단정", "전망성 낚시", "감정어").

# 출력 형식 (이 YAML만 출력, 다른 말·인사·설명 금지)
```yaml
penalties:
  - expr: "다 했네"          # 매칭 대상 문자열 또는 정규식
    type: regex             # phrase | regex
    tier: strong            # strong | medium | weak
    weight: 15
    근거: "조롱조 단정"        # 감점 메모에 쓰일 짧은 한글 라벨
    example: "반도체가 다 했네"  # 표본/현실 제목 중 이 표현이 걸리는 예
  - expr: "아우성"
    type: phrase
    tier: medium
    weight: 8
    근거: "감정 과장"
    example: "시장은 '고착화 우려' 아우성"
exclusions:
  - term: "충격"
    reason: "지진·사고 등 사실 문맥에 빈번해 단독 감점 시 오탐"
```

## 받은 뒤 우리가 할 일 (메모)
1. YAML을 검수해 오탐 위험 항목 제거·가중치 조정(정밀도 우선).
2. 채택본을 `src/objectivity.py`의 사전 구조로 옮긴다 — 이때 채점 코드가 **등급 가중치**를 반영하도록
   수정(현재는 매칭당 일괄 −10). 별도 설계·TDD로 진행.
3. `hits`(걸린 표현 + 근거) 집계 → **당일 감점 요약 메모**를 `articles-YYYY-MM-DD.json`에 추가하고
   아침 브리핑 3번에 노출.
