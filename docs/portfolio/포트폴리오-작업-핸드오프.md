  입장에서 생각하고 기획하는 PM이다.* — AI 실행력은 헤드라인이 아니라 **증거**로 배치.

- **정직성:** 실제 코드·문서·라이브에 있는 것만. **지어낸 성과 지표 금지.**

## 2. 서사 구조 — "기획 결정 6장면"

헤드라인(기획력) → 도입(200자) → **6장면** → 마무리(AI=도구·배운 것·다음).
각 장면 공통 미니 틀:

> **① 사용자 관점의 질문 → ② 내가 내린 결정 → ③ 트레이드오프(무엇을 포기했나)
> → ④ 그래서 무엇이 달라졌나(증거)**

AI는 **장면 6에서만 "도구"로 등장.** 헤드라인은 끝까지 기획력이 가져간다.

## 4. 증거 링크 규칙 (PDF·공개저장소)

- 핵심 증거는 **공개 blob URL**(§1 베이스 + 경로)로 클릭 검증되게.
- ASCII 경로(`src/objectivity.py` 등)는 **직접 blob 링크**. 한글/공백 경로는 **파일명 프로즈
  인용 + 저장소 링크**로 유도.
- 라이브 증명은 **라이브 서비스 URL**로. **저장소 로컬 상대경로 링크는 쓰지 않는다**(PDF 무의미).
- PDF 자기완결: 라이브 화면 **스크린샷**을 본문에 삽입(장면 3·6·마무리 중심) — 마지막 단계에 일괄.

## 6. 지금까지 쓴 초안 (헤드라인 + 도입)

```markdown
# 누구나 AI를 다루는 시대, 결과를 가르는 건 기획력이다

> 1인이 사용자 관점으로 기획하고, AI를 지휘해 **매일 아침 실제로 가동되는 서비스**까지 만든 기록
> — AI 뉴스 다이제스트

**이재혁** · 2026-07-10 · [라이브 서비스](https://daily-news-digest-service.vercel.app) · [저장소](https://github.com/jhyuk9776-jpg/-Daily-News-Digest-Service-)

---

AI로 무언가를 만드는 일은 이제 특별하지 않다. 프롬프트 몇 번이면 그럴듯한 결과물이 나온다.
그래서 오히려 변별력은 다른 곳에서 생긴다 — **무엇을, 왜, 누구를 위해 만들 것인가를 정하는
기획력**이다.

이 포트폴리오는 'AI 뉴스 다이제스트'라는 실제 가동 서비스를 만들며 내가 내린 **6개의 기획
결정**으로 그 기획력을 보여준다. 각 결정은 `사용자 관점의 질문 → 내가 내린 결정 →
포기한 것(트레이드오프) → 그래서 무엇이 달라졌나(증거)` 순서로 정리했다. AI는 이 과정에서
손이 빠른 실행 도구였을 뿐, **방향을 정하고 아닌 것을 걸러낸 것은 사람의 판단**이었다.

결과물은 매일 아침 사실 중심으로 요약된 뉴스가 이메일과 웹으로 발송되는, **지금도 돌아가는
서비스**다.
```

## 7. 아직 안 정한 것 (다음 세션 시작 시 결정)

- **헤드라인 문구** 확정 여부(더 세게/담백하게 조정 가능).
- **서술체 vs 존댓말체** — 현재 "~했다" 서술체 초안. "저는 ~했습니다"로 바꿀지.
- **장면 4 인용 방식** — 사례연구를 요약 인용할지, 링크만 걸지.

  ② **공개 저장소 URL**(§1)로 참조/검증(로컬 접근 불필요, PDF 링크와 동일해서 권장).
- **분리 권고:** 개인 취업 포트폴리오를 **공개된 제품 저장소 안에 두는 게 어색**할 수 있다.
  포트폴리오 산출물은 **별도 폴더/개인 저장소**로 옮기고, 이 제품 저장소는 증거를 **링크로만**
  참조하는 구성을 권한다. (제품 앱에 안 넣기로 한 것과 같은 이유.)
