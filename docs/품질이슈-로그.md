# 품질 이슈 로그

다이제스트 결과물에서 발견한 **품질 문제**를 축적한다. 런타임 실패(요약 0불릿·추출 실패·API 에러)는
`failures/failures-<date>.json`(코드 자동 기록)에 남고, 이 문서는 **사람이 눈으로 발견한 품질 결함**
(제목-내용 불일치, 분야 오분류, 홍보성 통과 등)을 모은다. 실패를 버리지 않고 자산으로 본다는 방침의 연장.

> 스키마: 각 이슈는 `ID · 발견일 · 유형 · 증상 · 사례 · 원인 가설 · 상태`. 해결되면 상태를 `해결`로 바꾸고
> 커밋 해시를 적는다.

## ISSUE-001 — 제목과 요약 내용이 다른 기사 존재

- **발견일:** 2026-07-06 (이재혁)
- **유형:** 제목-내용 불일치 (title/content mismatch)
- **상태:** 🟡 수정 반영(관찰 중) — 2026-07-06
- **2026-07-06 수정:** 근본 원인이 `extract_body`의 추천위젯 오추출로 확인됨. 도메인별 본문 선택자(`SITE_SELECTORS`)
  + 검증 가드(`looks_like_body`: 날짜 3개↑·제목불일치 기각)로 진짜 본문만 추출. 실측에서 한국경제 본문 정상 복원.
  다음 다이제스트에서 재발 여부 관찰.

### 증상
다이제스트의 기사 **제목**과 그 아래 **요약 불릿**이 서로 다른 사건을 가리키는 경우가 있다.
불릿이 여러 기업·주제를 잡탕으로 담아, 제목이 말하는 기사와 무관한 사실이 섞여 나온다.

### 사례 (2026-07-06 다이제스트, 경제)
| 제목 | 실제 요약 불릿 내용 | 불일치 |
|---|---|---|
| "2500원 더 내고 3일 기다렸는데…올영, **제주도민** 설움 덜었다" | 올리브영 **택스리펀드(외국인)** 데이터 분석, 외국인 관광객 재방문, 오프뷰티 쿠팡이츠 입점 | 제목=제주도민 / 내용=외국인 택스리펀드 |
| "과일만 1kg?"…신세계푸드, 3만원대 **'케이크' 출시** | 신세계푸드 **식품안전 상생교육**, 롯데마트 과일 매출 증가 | 제목=케이크 출시 / 내용=식품안전 교육 |
| "**손흥민** 찰칵 세리머니…월드컵 홀린 아틀라스" | 현대차 로봇 **아틀라스 축구 학습 메이킹필름**, 폭스바겐 어린이 그림대회 | 제목=손흥민/월드컵 / 내용=로봇·그림대회 |

### 원인 가설
- 불릿이 **여러 기사의 사실을 섞은 잡탕** 형태 → 요약 대상 본문이 그 기사 하나가 아니라
  **여러 기사를 나열한 종합/미리보기 페이지**이거나, 추출된 본문 URL이 제목 기사와 어긋났을 가능성.
- 또는 요약 폴백 체인이 **같은 클러스터의 다른 매체·다른 기사 본문**으로 넘어가 제목과 본문이 분리됐을 가능성
  (`extract.iter_contents` → `summarize.summarize_item` 폴백 오케스트레이션 점검 필요).
- 공통점: 한국경제 연성기사(생활·유통)에서 두드러짐 → 해당 매체 본문 페이지가 묶음형일 수 있음.

### 다음 액션(제안, 미착수)
- 위 3개 링크의 **실제 본문 vs 추출 텍스트**를 대조해 어느 단계에서 어긋나는지 특정.
- 추출 단계에서 "제목 기사 본문"만 정확히 잡히는지(오추출·묶음페이지) 검증.

## ISSUE-002 — 과도한 요약: 본문의 핵심 수치가 불릿에서 누락

- **발견일:** 2026-07-06 (이재혁, 대상은 07-05 다이제스트)
- **유형:** 과요약 / 핵심 사실(특히 수치) 손실
- **상태:** 🟡 수정 반영(관찰 중) — 2026-07-06
- **2026-07-06 수정:** ISSUE-001과 같은 뿌리(추천위젯 오추출). 본문 우선 순서(`iter_contents`가 검증 통과 본문을
  RSS 요약보다 먼저 사용)로 원문 수치 확보 + 요약 프롬프트에 "숫자·비율·금액·기간·기관명 보존" 규칙 추가.
  실측 재검증: 식량가격지수 기사 본문에서 `130.3 포인트`·`0.3%` 복원 확인.

### 증상
불릿이 **제목을 거의 그대로 되풀이**하는 수준으로 압축돼, 본문에 있는 **구체 수치·하락폭·기간** 등
핵심 근거가 빠진다. "증거 우선(숫자·기관·인용)"이 서비스의 핵심 차별점인데 정작 숫자가 사라진다.

### 사례 (2026-07-05 다이제스트, 경제)
| 제목 | 실제 불릿 | 빠진 것 |
|---|---|---|
| "세계 식량가격지수, 2개월 연속 하락…곡물·유제품·설탕 가격↓" | "세계 식량가격지수가 2개월 연속 하락했으며, 곡물·유제품·설탕 가격이 내렸다." | 지수값·**하락폭(%)**·수치. 본문엔 있으나 요약에서 누락 |

→ 불릿 1개가 제목의 동어반복. 본문의 정량 정보(얼마나 내렸는지)를 담지 못함.

### 원인 가설
- 요약 프롬프트가 "간결·불릿"을 강조하다 보니 **정량 정보 보존 지시가 약함** → 모델이 제목 수준으로 압축.
- 짧은 RSS 요약만으로 요약했을 때(본문 미추출) 수치가 원문에 아예 없었을 가능성 → 추출 트리거 조건 점검.
- 불릿 수 상한/길이 압박이 수치를 먼저 버리게 만들 가능성.

### 다음 액션
- ISSUE-001과 함께 **요약 품질 개선 브레인스토밍**에서 다룬다(2026-07-06 착수).
- 방향 후보: 요약 프롬프트에 "숫자·비율·금액·기간은 반드시 보존" 규칙 추가 / 본문 추출을 수치 포함 기사에 더 적극 적용 / 불릿 최소 정보량 기준.

## ISSUE-004 — 같은 매체·같은 날 같은 사건 기사가 중복 선별

- **발견일:** 2026-07-08 (이재혁, 대상은 07-08 다이제스트 경제)
- **유형:** 중복 제거 실패 (dedup miss) — 제목 기반 클러스터링의 구조적 한계
- **상태:** 📌 한계로 기록 (이번 사이클 미수정, 2026-07-08 결정) — 본문/엔티티 기반 중복제거는 Phase 2 후보

### 증상
같은 매체가 같은 날 **같은 사건을 다른 제목으로 여러 건** 내보내면, 제목 유사도 클러스터링
(`cluster_articles`, `SequenceMatcher ≥ 0.6`)이 이를 별도 클러스터로 잡아 둘 다 선별한다.

### 사례 (2026-07-08 다이제스트, 경제 · 모두 연합뉴스)
| 사건 | 중복 기사 A | 중복 기사 B |
|---|---|---|
| 최저임금 협상 | "최저임금 노 1만1천450원·사 1만460원…격차 990원으로(종합)" | "노동계 1만1천500원·경영계 1만440원…최저임금 5차 수정안" |
| 코스피 급락 | "코스피, 급락에 사이드카·서킷 발동…4.9%↓ 7,600선(종합)" | "코스피, 4.9% 급락 7,600선…코스닥도 올해 최저치" |

### 원인 (실측)
최저임금 두 제목의 정규화 후 유사도는 **0.222**(임계 0.6), 공유 토큰은 `{'최저임금'}` 하나뿐
(자카드 0.071). 숫자·표현이 완전히 달라 제목만으로는 같은 사건임을 판별할 신호가 없다.
임계값을 낮추면 무관한 기사까지 오병합되는 부작용이 커 **제목 기반으로는 해결 불가**.

### 판단 (2026-07-08)
- 이번 사이클에선 **수정하지 않고 한계로 기록만** 한다(무료 웹 출시 우선).
- 실제 해결은 **본문/엔티티 기반 중복 제거**(같은 인물·수치·기관을 공유하는지)나 **LLM 중복 판정**이
  필요 → Phase 2 후보(로드맵 §8 "1차 출처/엔티티" 계열과 함께).

### 추가 조사 (2026-07-10, 이재혁) — "어휘/빈도로는 불가"를 실측으로 증명
07-08엔 임계값 하나로 판단했지만, 07-10엔 대안을 **실제 구현·테스트**해 한계를 정량 확인했다.

- **실측 대상:** 로컬 `raw/2026-07-03.json` 경제 369건. 구 방식(SequenceMatcher≥0.6) = 350 클러스터.
- **시도한 대안과 결과(과잉병합):**
  | 방식 | 클러스터 | 최대 병합군 |
  |---|---|---|
  | 토큰 겹침(2개↑) + 연결요소 병합 | 254 | **56건** 🔴 |
  | 문서빈도(df) 가중, 드문 단어만 신뢰 | 306~ | **99건** 🔴 |

  56건 블롭 예: "주유소 경유 가격" + "삼성 영남 312조 투자" + "AI 유리기판"이 `코스피·투자·영남`
  같은 흔한 단어를 매개로 줄줄이 오병합.
- **핵심 통찰(왜 불가):** 좋은 연결어와 오탐 유발어가 **같은 문서빈도 구간**에 섞여 있다.
  `홈플러스(df=8, 좋은 연결어)` vs `코스피(df=9, 오탐 유발)` — 빈도가 거의 같아 어떤 컷오프로도 못 가름.
  `영남(df=2)`은 드문데도 서로 다른 대기업 투자발표를 오병합(=드묾≠좋은 신호). `코스피`는 연결어이자
  동시에 오탐 원인. → **빈도만으로는 "같은 사건인가"를 못 판정. 의미(semantic)가 필요.**
- **방법 비교(로컬 제약 하):** 로컬 임베딩(sentence-transformers)만이 광주 3건 완전 병합(3→1)을 오탐 없이
  달성 가능. 단 torch 등 무거운 의존성(침습도 높음). LLM 그룹핑은 정확하나 유료 API라 이번 후보 제외.
- **방향 확정:** M4(출시 대기)에선 **안전 어휘(별칭맵 등)로 부분 개선(광주 3→2)**만, 완전한 의미 병합(3→1)은
  **Phase 2 로컬 임베딩**으로. 상태는 📌 한계 기록 유지. 상세는 `docs/portfolio/2026-07-10-중복제거-사례연구.md`.

## ISSUE-005 — 같은 사건이 카테고리 경계를 넘어 중복 선별 (교차 중복)

- **발견일:** 2026-07-10 (이재혁, 대상은 07-10 다이제스트)
- **유형:** 중복 제거 실패 (dedup miss) — 카테고리별 독립 클러스터링의 구조적 한계
- **상태:** 🟡 방향 결정·미구현 (2026-07-10) — 안전한 해법(전역 seen-set) 설계 완료, M4 안전 우선으로 승인 후 구현

### 증상
같은 사건이 서로 다른 카테고리 피드에 올라오면, 각 분야에서 **각각 대표로 선별**돼 다이제스트에 중복 노출된다.

### 사례 (2026-07-10 다이제스트)
| 사건 | 경제 | IT/테크 |
|---|---|---|
| 광주 군공항 토지거래허가구역 지정 | "광주 군공항 인근 토지거래허가구역 지정…14일부터 효력(종합)" (연합뉴스) | "호남 반도체 산단 투기 차단…광주 군공항 일대 토지거래허가구역 지정" (전자신문) |

### 원인 (코드 구조)
`select()`가 `by_cat` 루프로 **카테고리별로만** `cluster_articles`를 돌린다(`src/curate.py`). 분야가 다르면
클러스터링 비교 자체가 일어나지 않아, 같은 사건이 여러 분야에 그대로 남는다.

### 방향 (설계 완료, 2026-07-10)
- **분야를 우선순위 순서로 처리 + 전역 seen-set:** `경제 > 사회 > 세계 > IT/테크` 순으로 선별하며, 이미 채택한
  사건 대표를 전역 집합에 기록. 뒤 분야에서 같은 사건이면 건너뛰고 다음 기사로 **백필**(구멍 방지).
- **제품 결정(승인됨):** 교차 시 **우선순위 높은 분야가 사건을 차지**한다(광주 토허구역=부동산/경제 성격 → 경제).
- **안전성:** 교차 비교는 분야 대표(상위 ~2건)끼리만 → 비교 물량이 작아 오탐 표면이 작다.
- **미구현 이유:** M4는 안전 우선. 패턴 A와 함께 승인 후 구현 예정.

## CONSTRAINT-001 — 국내 주요 매체 다수가 공개 RSS 폐지 (수집 제약)

- **유형:** 수집 인프라 제약(실패 축적)
- **상태:** 📌 기록 — 스크래핑 수집은 후속 과제

### 사실
매체 확장을 위해 RSS를 실측한 결과, 상당수 매체가 **공개 RSS를 폐지**했다.

| 매체 | RSS 상태(2026-07-06) |
|---|---|
| 연합뉴스 | ✅ 분야별 정상(economy/society/international) → 추가함 |
| 한겨레 | ✅ 분야별 정상 → 추가함 |
| 뉴시스 | ✅ 정상(통신사 대안 후보, 이번엔 미채택) |

| YTN | ❌ 폐지 |
| MBC(imnews) | ❌ 폐지 |

## ISSUE-007 — 한국경제 RSS 수집이 gzip 처리 결함으로 조용히 실패

- **발견일:** 2026-07-11 (이재혁·Claude, 클러스터 분포 측정 중)
- **유형:** 수집 회귀 (silent collection failure) — feedparser 내장 HTTP의 gzip 처리 결함
- **상태:** 🟢 해결 (2026-07-11, `fetch.py` 바이트 직접수신 폴백)

### 증상
`feedparser.parse(url, agent=USER_AGENT)`가 한국경제 피드에서 항상 `not well-formed (invalid token)`(2:1326)로 bozo·엔트리0을 반환 → 수집 실패. `failed_sources`에 기록되나 파이프라인은 그대로 진행해 **객관성 1위 매체가 조용히 누락**. media.json상 한국경제 `last_seen`가 2026-07-09에 멈춘 것과 일치(07-10경부터 누락).

### 원인 (실측)
- feedparser 내장 HTTP는 `Accept-Encoding: gzip`을 보내는데, 한국경제 응답의 gzip 처리에서 깨진 바이트가 나와 XML 파싱 실패.
- 반면 `urllib`로 gzip 미요청 평문 수신 → 정상 XML(50건). 즉 UA 문자열이 아니라 **HTTP 계층(압축) 문제**.
- 부수 확인: 한국경제는 전체 Chrome UA를 **403**으로 차단하고, 짧은 `Mozilla/5.0`은 허용.

### 조치
- `fetch_feed`에 폴백 추가: 1차 파싱이 bozo·엔트리0이면 `_fetch_bytes(url)`(urllib, gzip 미요청, 짧은 `Mozilla/5.0` UA)로 바이트를 직접 받아 재파싱. 정상 피드는 경로 불변(폴백은 실패 시에만).
- 폴백 단위 테스트 3종 추가(`tests/test_fetch_gzip_fallback.py`). 전체 스위트 117 통과, 한국경제 실데이터 50건 복원 확인.

### 남은 관찰
- 다른 매체도 같은 방식으로 언제든 깨질 수 있음 → 폴백이 일반 안전망 역할. `failed_sources`가 계속 쌓이면 알림 강화 검토.
