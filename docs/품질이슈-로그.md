# 품질 이슈 로그

다이제스트 결과물에서 발견한 **품질 문제**를 축적한다. 런타임 실패(요약 0불릿·추출 실패·API 에러)는
`failures/failures-<date>.json`(코드 자동 기록)에 남고, 이 문서는 **사람이 눈으로 발견한 품질 결함**
(제목-내용 불일치, 분야 오분류, 홍보성 통과 등)을 모은다. 실패를 버리지 않고 자산으로 본다는 방침의 연장.

> 스키마: 각 이슈는 `ID · 발견일 · 유형 · 증상 · 사례 · 원인 가설 · 상태`. 해결되면 상태를 `해결`로 바꾸고
> 커밋 해시를 적는다.

## ISSUE-001 — 제목과 요약 내용이 다른 기사 존재

- **발견일:** 2026-07-06 (이재혁)
- **유형:** 제목-내용 불일치 (title/content mismatch)
- **상태:** 🟡 수정 반영(관찰 중) — 2026-07-06
- **2026-07-06 수정:** 근본 원인이 `extract_body`의 추천위젯 오추출로 확인됨. 도메인별 본문 선택자(`SITE_SELECTORS`)
  + 검증 가드(`looks_like_body`: 날짜 3개↑·제목불일치 기각)로 진짜 본문만 추출. 실측에서 한국경제 본문 정상 복원.
  다음 다이제스트에서 재발 여부 관찰.

### 증상
다이제스트의 기사 **제목**과 그 아래 **요약 불릿**이 서로 다른 사건을 가리키는 경우가 있다.
불릿이 여러 기업·주제를 잡탕으로 담아, 제목이 말하는 기사와 무관한 사실이 섞여 나온다.

### 사례 (2026-07-06 다이제스트, 경제)
| 제목 | 실제 요약 불릿 내용 | 불일치 |
|---|---|---|
| "2500원 더 내고 3일 기다렸는데…올영, **제주도민** 설움 덜었다" | 올리브영 **택스리펀드(외국인)** 데이터 분석, 외국인 관광객 재방문, 오프뷰티 쿠팡이츠 입점 | 제목=제주도민 / 내용=외국인 택스리펀드 |
| "과일만 1kg?"…신세계푸드, 3만원대 **'케이크' 출시** | 신세계푸드 **식품안전 상생교육**, 롯데마트 과일 매출 증가 | 제목=케이크 출시 / 내용=식품안전 교육 |
| "**손흥민** 찰칵 세리머니…월드컵 홀린 아틀라스" | 현대차 로봇 **아틀라스 축구 학습 메이킹필름**, 폭스바겐 어린이 그림대회 | 제목=손흥민/월드컵 / 내용=로봇·그림대회 |

### 원인 가설
- 불릿이 **여러 기사의 사실을 섞은 잡탕** 형태 → 요약 대상 본문이 그 기사 하나가 아니라
  **여러 기사를 나열한 종합/미리보기 페이지**이거나, 추출된 본문 URL이 제목 기사와 어긋났을 가능성.
- 또는 요약 폴백 체인이 **같은 클러스터의 다른 매체·다른 기사 본문**으로 넘어가 제목과 본문이 분리됐을 가능성
  (`extract.iter_contents` → `summarize.summarize_item` 폴백 오케스트레이션 점검 필요).
- 공통점: 한국경제 연성기사(생활·유통)에서 두드러짐 → 해당 매체 본문 페이지가 묶음형일 수 있음.

### 다음 액션(제안, 미착수)
- 위 3개 링크의 **실제 본문 vs 추출 텍스트**를 대조해 어느 단계에서 어긋나는지 특정.
- 추출 단계에서 "제목 기사 본문"만 정확히 잡히는지(오추출·묶음페이지) 검증.
- 재발 방지책은 별도 브레인스토밍으로 설계(임시 처치 금지).

## ISSUE-002 — 과도한 요약: 본문의 핵심 수치가 불릿에서 누락

- **발견일:** 2026-07-06 (이재혁, 대상은 07-05 다이제스트)
- **유형:** 과요약 / 핵심 사실(특히 수치) 손실
- **상태:** 🟡 수정 반영(관찰 중) — 2026-07-06
- **2026-07-06 수정:** ISSUE-001과 같은 뿌리(추천위젯 오추출). 본문 우선 순서(`iter_contents`가 검증 통과 본문을
  RSS 요약보다 먼저 사용)로 원문 수치 확보 + 요약 프롬프트에 "숫자·비율·금액·기간·기관명 보존" 규칙 추가.
  실측 재검증: 식량가격지수 기사 본문에서 `130.3 포인트`·`0.3%` 복원 확인.

### 증상
불릿이 **제목을 거의 그대로 되풀이**하는 수준으로 압축돼, 본문에 있는 **구체 수치·하락폭·기간** 등
핵심 근거가 빠진다. "증거 우선(숫자·기관·인용)"이 서비스의 핵심 차별점인데 정작 숫자가 사라진다.

### 사례 (2026-07-05 다이제스트, 경제)
| 제목 | 실제 불릿 | 빠진 것 |
|---|---|---|
| "세계 식량가격지수, 2개월 연속 하락…곡물·유제품·설탕 가격↓" | "세계 식량가격지수가 2개월 연속 하락했으며, 곡물·유제품·설탕 가격이 내렸다." | 지수값·**하락폭(%)**·수치. 본문엔 있으나 요약에서 누락 |

→ 불릿 1개가 제목의 동어반복. 본문의 정량 정보(얼마나 내렸는지)를 담지 못함.

### 원인 가설
- 요약 프롬프트가 "간결·불릿"을 강조하다 보니 **정량 정보 보존 지시가 약함** → 모델이 제목 수준으로 압축.
- 짧은 RSS 요약만으로 요약했을 때(본문 미추출) 수치가 원문에 아예 없었을 가능성 → 추출 트리거 조건 점검.
- 불릿 수 상한/길이 압박이 수치를 먼저 버리게 만들 가능성.

### 다음 액션
- ISSUE-001과 함께 **요약 품질 개선 브레인스토밍**에서 다룬다(2026-07-06 착수).
- 방향 후보: 요약 프롬프트에 "숫자·비율·금액·기간은 반드시 보존" 규칙 추가 / 본문 추출을 수치 포함 기사에 더 적극 적용 / 불릿 최소 정보량 기준.

## CONSTRAINT-001 — 국내 주요 매체 다수가 공개 RSS 폐지 (수집 제약)

- **발견일:** 2026-07-06 (매체 확장 작업 중 실측)
- **유형:** 수집 인프라 제약(실패 축적)
- **상태:** 📌 기록 — 스크래핑 수집은 후속 과제

### 사실
매체 확장을 위해 RSS를 실측한 결과, 상당수 매체가 **공개 RSS를 폐지**했다.

| 매체 | RSS 상태(2026-07-06) |
|---|---|
| 연합뉴스 | ✅ 분야별 정상(economy/society/international) → 추가함 |
| 한겨레 | ✅ 분야별 정상 → 추가함 |
| 뉴시스 | ✅ 정상(통신사 대안 후보, 이번엔 미채택) |
| KBS | ❌ 폐지(에러 페이지 리다이렉트) |
| YTN | ❌ 폐지 |
| MBC(imnews) | ❌ 폐지 |
| 중앙일보(joins) | ❌ "서비스 종료 안내" |
| 블로터 | ❌ 피드 없음(403/404) |

### 판단
- 구글뉴스 RSS 대체는 **비추천**: 링크가 `news.google.com` 리다이렉트라 도메인별 본문 추출 불가 + 검색결과 스팸 혼입(실측 시 KBS 결과에 스팸 제목).
- 폐지 매체를 넣으려면 **스크래핑(목록 페이지 파싱)** 필요 → 매체별 목록·본문 선택자 2개 유지, 메타 부실, robots/약관 민감. **개인용·RSS 범위 밖 → 후속 별도 설계**로 미룸.
- IT/테크는 추가 가능한 신규 매체가 없어 전자신문·ZDNet 유지.

## ISSUE-003 — 객관성 감점: `…`(말줄임표) 대량 오탐

- **발견일:** 2026-07-06 (이재혁·Claude, 변별력 그물망 backfill 검증 중)
- **유형:** 감점 사전 오탐 (false positive) — 구조적 낚시 신호
- **상태:** 🟢 해결 (커밋 penalties.yaml `…` 제거)

### 증상
객관성 변별력 개편(감점 밀도 전환) 후 첫 backfill에서, 07-01 486건 중 **294건(60%)**이
`(\?!|!!+|…)` "과도한 문장부호"에 걸려 감점됐다. 감점 point의 절대다수(882/약 1000pt)가 이
한 신호에서 나와, 매체 변별력이 사실상 "누가 제목에 `…`를 더 쓰나"를 재는 노이즈가 됐다.

### 원인
`…`는 한국 뉴스 제목의 **표준 연결·생략 부호**다(예: "무역수지 361억달러 흑자**…**사상 첫
300억 상회"). 선정성이 아니라 정상 문체인데 낚시 신호로 잡았다.

### 조치
`penalties.yaml` 구조 신호에서 `…` 제거(`?!`, `!!+` 중복 감탄/의문만 유지). 재backfill 결과
07-01 감점 298→17건으로 정상화, 밀도 분포가 실제 평가·조롱·낚시 표현 기준으로 재편(한국경제
0.0 ~ 전자신문 388.9/1k). 한국경제 최저(가장 객관적)로 기존 직관과 일치.

### 잔여 캘리브레이션 후보(미해결, 데이터 더 쌓은 뒤 판단 — observe-only라 위험 0)
- `\?$`(물음표 종결, medium 8): 정당한 질문형 제목 오탐 여지. weight/포함여부 재검토.
- **인용 밀도(②) 교란:** 매체별 RSS 요약 제공 여부에 좌우됨(한국경제 summary 빈 값 → 인용 0).
  "귀속 자세"가 아니라 "요약 제공 여부"를 재고 있을 수 있음.
- **교차 이상치(④) 정의:** 전량 raw는 매체별 제목 표현이 달라 대부분 단독 클러스터로 잡힘 →
  이상치가 "hit 있는 기사"와 거의 동의어가 됨. 클러스터링 기준 재검토 필요.
- 표본이 3일(90~486건/매체)로 작음 → 신규 매체(연합·한겨레) 포함해 며칠 축적 후 값 확정.
