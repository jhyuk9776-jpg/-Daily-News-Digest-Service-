# 선별 재설계 — 본문 채점 대표 선정 + 코어단어 주제 + 선택률 평판

작성: 2026-07-13 · **개정: 2026-07-14 (선별 흐름 전면 상세화)** (이재혁 · Claude) · 상태: **설계 확정, 구현 중(Phase 0 완료)**

## 배경

기존 선별은 **제목 유사도(교차검증) + 매체 수동 우선순위**로 대표를 뽑고 본문은 뽑은 뒤에만 읽었다.
- 감점 밀도 시스템은 "1순위 매체만 보자"는 목적이었으나 **순위 높은 매체가 꼭 잘 쓴 건 아님**을 발견.
- 매체 우선순위(연합뉴스 앵커)가 **단일 매체 쏠림**을 만듦(07-08~11 연합뉴스 14/16 독점).
- 관련성 게이트 부재로 잡토픽 오분류(ISSUE-006). 제목 유사도만으론 같은 사건인데 못 묶는 경우 발생.

## 목표

여러 매체가 보도한 사건을 후보로 삼고, **본문을 우리 기준(객관성 + 근거성)으로 채점해 가장 잘 쓴 한 편**을 대표로 골라 요약한다. 그 선택 결과로 매체 평판(선택률)을 축적하고, 코어단어로 "그날의 주제"를 관찰 축적한다.

## 확정된 핵심 결정

1. **객관성 잣대** = 룰 기반 감점(penalties.yaml)을 **제목+본문 전체**에 적용. LLM 아님(일관·저비용·결정론).
2. **근거성 잣대** = 본문 **문장 커버리지**(근거 든 문장 비율). 길이에 안 휘둘리고 0~1 상한.
3. **후보 풀** = 교차검증(2곳↑) 클러스터 우선, 부족분은 단독으로 **매체 우선순위 라운드로빈 백필**.
4. **선택은 본문 채점만으로**(매번 신선하게). 매체 평판은 선택을 **몰지 않음**.
5. **선택률 = 승리 ÷ 등장**(정규화). 원시 횟수 금지(통신사 물량 편향).
6. **매체 캡 없음** — 관찰 후 판단.
7. **자동 제외 없음** — 사람 검토 플래그만.
8. **감점 밀도 유지** — 브리핑 지표 + **매체 우선순위(백필/tiebreak) 파생용**으로 계속 축적.
9. **매체 수동 우선순위 폐지 유지** — 우선순위는 sources.yaml이 아니라 **media.json 감점 밀도에서 런타임 파생**(density 낮을수록 상위). 수동 서열은 넣지 않음.
10. **코어단어** — 그날 분야별 제목에서 **자동 추출**. 클러스터링 보강 + 정렬 tiebreak + "그날의 주제" 관찰에 사용. 수동 시드 없음.
11. **길이 필터** — 본문 300~1500자만 후보. 이탈분은 관찰 로그로 기록(임계값 근거 축적).

## 선별 흐름 (확정본)

분야별로 아래 순서로 진행한다.

### 1. 2단계 클러스터링
- **1차(같은 매체 내부)**: 제목 유사도(≥0.6) + 코어단어 공유로 묶고, 한 매체가 같은 사건을 여러 번 보도하면 **가장 최근 1건만 유지**(나머지는 개수로만 집계). 예: 한국경제 코스피 3건 → 최근 1건 대표, count=3.
- **2차(매체 간)**: 1차 정리된 매체별 대표들을 교차 클러스터링 → 교차검증(등장 매체 2곳↑) / 단독 분리.

### 2. 코어단어 집계·주제·가중치 갱신 (길이 필터 전, 관찰 기록)
클러스터마다 기록:
- 매체별 합산 기사 수(예: 한국경제 3, 매일경제 3, SBS 2), **등장 매체 수**(3), **총 기사 수**(8), 포함 코어단어.
- **코어단어 점수 = 등장 매체 수 + (총 기사 수 × 0.2)** — 매체 다양성이 물량보다 객관적이라 매체 수를 주가중.
- **오늘의 주제 = 코어단어 점수 top-3** → `scores/topics-<date>.json`에 기록(observe, 활용은 나중).
- **코어단어 가중치 갱신**: top-3에 순위 점수(1위 5·2위 3·3위 1)를 감쇠 상수 α로 완만 반영 — `weight += 점수 × α`(α=0.1). `scores/core-word-weights.json`에 누적·영속.
  - 저장소는 **빈 상태로 시작**. 첫날 top-3가 초기값. 첫날 정렬 tiebreak은 가중치 부재로 무동작(부트스트랩), 둘째 날부터 작동.
  - 날짜 멱등: 같은 날짜 재실행은 가중치 이중 반영 금지.

### 3. 길이 필터 — 대표 선정 시점에 적용 (비용 통제)
본문 **300~1500자**만 유효. 단, 길이 판정은 본문 추출이 필요하므로 **모든 후보가 아니라 선정 클러스터 멤버 추출 시점(6단계)에 적용**한다(전 후보 추출 = 비용 폭증 방지). 범위 이탈 멤버는 대표 후보에서 제외하고 `scores/length-excluded-<date>.json`에 기록(source·link·length·category). 클러스터에 유효 길이 멤버가 없으면 그 클러스터는 탈락(선정 부족분은 백필로 보충). 코어단어 작업(제목 기반, 저렴)은 앞단(2단계)에 그대로 둔다.

### 4. 클러스터 정렬
1순위 **등장 매체 수 ↓** → 동수면 **제목에 코어단어 포함** 클러스터 우선 → 그것도 같으면 **제목의 코어단어 가중치 합 ↓**(2단계에서 갱신된 값). 최종 동점은 최신순.

### 5. 상위 10 선정 + 백필
전 분야 상한 **10**. 교차검증 클러스터가 10 미만이면 남은 자리를 **단독 기사**로 채우되, 매체 우선순위(density 낮을수록 상위) **상위 3개 매체만 라운드로빈**(1위 1편 → 2위 1편 → 3위 1편 → 다시 1위 1편 → … 10위까지). 각 매체의 최신 단독부터.

### 6. 대표 선정 (본문 채점표)
선정된 클러스터마다 멤버 매체 본문을 추출 → **길이 필터(300~1500, §3) 적용** → 통과 멤버를 2칼럼으로 채점, **총합 최고 = 대표**:
- **문장 객관성** = `1 − min(감점,45)/45` (감점 낮을수록 1). 감점 = `body_objectivity(제목,본문)`.
- **단어 선택(근거성)** = 문장 커버리지(근거 든 문장 비율, 0~1).
- **총합 = 0.6 × 객관성 + 0.4 × 근거성**.
- **동점 tiebreak** = 매체 우선순위(density) 높은 쪽.
- 단독 클러스터는 그 한 편이 대표(채점 불필요, 다만 길이 필터는 통과해야 함).
- 본문 추출은 **선정된 클러스터로 이동**(현재는 요약 단계). 비용 통제.

### 7. 요약 + 평판 갱신
대표를 요약("외 관련 N건" 링크 유지). 교차검증 클러스터마다 멤버 매체 `appear+1`, 대표 매체 `win+1`.

## 본문 채점 함수 (Phase 0 완료)

- `objectivity.body_objectivity(body, title="") -> {points, hits, score}` — 제목+본문 전체 감점. scope:title 룰은 title="" 이면 미발화. 홍보성 평가어 `(성과|…).{0,3}입증` 등 기존 룰이 감점 1호 포착 확인.
- `objectivity.body_richness(body) -> float` — (참고용, 초기 밀도식) 유지하되 대표 채점의 근거성 칼럼은 **문장 커버리지**로 확정. → Phase 1에서 `sentence_coverage(body)` 신설.
- `curate.evidence_signals(text) -> int` — 숫자·%·기관·인용·기간 신호(문장 커버리지가 재사용).
- 라벨 검증(Phase 0 실측): 예시1호 감점 0·커버리지 낮음(객관·통과), 감점1호 감점 8(홍보성 포착).

## 코어단어 (신규 모듈)

- **추출**: 그날 분야별 제목에서 빈출 고유명사/명사(형태소 없이 정규식 `[가-힣A-Za-z0-9]{2,}` + 불용어 제거, 2개 분야↑ 또는 빈도≥2 등장 토큰). 유지비 0.
- **가중치 저장**: `scores/core-word-weights.json` `{단어: 가중치}`. 미등록 = 1. 매일 top-3로 갱신(§흐름 2).
- **주제 기록**: `scores/topics-<date>.json` `{date, topics:[{word, score, media, articles}]}` top-3.

## 평판 되먹임 (media.json 스키마 확장)

기존 `update_media_scores`(감점 밀도, 날짜 멱등)와 동일 패턴으로 선택률 누적:
```
"한국경제": { ...density 필드..., "appear_total": N, "win_total": M, "selection_rate": M/N }  # N=0이면 null
```
- 데이터 = 본문 선별의 부산물(추가 채점 0). 집계 = 교차검증 선정 클러스터만. 단독 제외.
- `update_selection_rates(store, daily_stats, date)`: 날짜 멱등 + 누적.
- 매체 우선순위(백필/tiebreak)는 `compute_ranks(store)`(기존, density 오름차순)를 재사용해 런타임 파생.

## 구현 순서

- **Phase 0 — 룰 재점검** ✅ 완료: body_objectivity(제목+본문)·body_richness·evidence_signals, 라벨 2개 검증. 커밋 5개, 123 테스트 OK.
- **Phase 1 — 채점 함수**: `sentence_coverage(body)` 신설 + 대표 채점표 `representative_score(title, body)`(0.6/0.4). 라벨로 검증.
- **Phase 2 — 코어단어**: 추출·점수(매체수+기사수×0.2)·주제 top-3 기록·가중치 파일 갱신(5/3/1×α, 날짜 멱등).
- **Phase 3 — 클러스터링 개편**: 2단계(매체 내부 최근1건 → 매체 간), 코어단어 공유 병합, 정렬(매체수→코어단어 포함→가중치합).
- **Phase 4 — 선정·백필·대표**: 상한 10, density 3위 라운드로빈 백필, 길이 필터(300~1500)+이탈 로그, 대표 채점, 파이프라인 배선(추출 이동).
- **Phase 5 — 선택률 평판**: media.json 스키마 + update_selection_rates + 브리핑 표시.
- **Phase 6 — 기자 블랙리스트 재통합·동작 확인**.
- 각 Phase TDD. 전체 회귀(현재 123 테스트) 유지.
