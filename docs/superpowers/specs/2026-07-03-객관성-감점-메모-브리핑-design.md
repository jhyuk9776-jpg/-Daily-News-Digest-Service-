# 설계: 객관성 감점 등급화 + 당일 감점 메모 + 브리핑 통합

작성일: 2026-07-03
작성: 이재혁
상태: 설계 확정 대기(사용자 검토 후 writing-plans)

## 1. 목적

매체 객관성 점수(observe-only)의 **판별력**을 높이고, 매일 아침 브리핑 때
**당일 감점 결과("무엇이·왜 감점됐나")와 누적 평판**을 함께 받는다.

배경: 현재 감점 사전은 정확 문구 4개뿐이라 거의 발동하지 않아 대부분 100점(판별력 부재).
로드맵 §6(매체 객관성 동적 점수), 결정 로그 1.3(객관적 서술 최우선), AI_CONTEXT §6.

## 2. 범위 / 비범위

- **범위:** ① 감점 사전 등급화(strong/medium/weak) 반영, ② 당일 감점 메모 집계,
  ③ `objectivity-score` 스킬 강화(fetch 폴백 + 메모·누적 출력), ④ `daily-briefing`이 이를 호출.
- **비범위:** 랭킹/선별 반영(계속 observe-only), raw 영구 보존·소급 재채점(D안 — 이후),
  LLM 실시간 채점(2번안 — 기각), cron 자동 채점(수동 유지).

## 3. 결정된 방향(브레인스토밍 결과)

| 항목 | 결정 |
|---|---|
| 감점 기준 형태 | 확장된 **결정적 사전/룰**(키워드·정규식). AI는 설계 시 1회만 사용 → 운영 비용 0 유지 |
| 사전 소스 | `docs/prompts/감점사전-결과-초안.yaml` 검수본(적용 22·관찰 3·제외 10) |
| 저장 방식 | **A안**: `articles-YYYY-MM-DD.json`(감점분) + `media.json`(누적) 파일 보존 |
| 스킬 구조 | **독립 스킬**(`objectivity-score`) + `daily-briefing`이 호출("연결" = 호출 스텝 1줄) |
| 관찰 후보 | 다더니·이러다·극적은 `apply:false`로 채점 미적용·기록만 |

## 4. 아키텍처 / 컴포넌트

### 4.1 감점 사전(설정) — `penalties.yaml`(신규, 루트)
`sources.yaml`·`limits.yaml`과 같은 루트 설정 파일로 승격. 검수본 YAML을 옮긴다.
스키마: `penalties[]`(expr, type[phrase|regex], tier, weight, 근거, example),
`observe_candidates[]`(+apply:false), `exclusions[]`(term, reason).

### 4.2 채점기 — `src/objectivity.py`(수정)
- **사전 로더 추가**: `load_penalties(path) -> (active, observe, exclusions)`.
  파일 없으면 현행 하드코딩 시드로 폴백(하위호환).
- `objectivity_score()` 변경: 매칭당 **weight 합산 감점**(현재 일괄 −10 대체).
  - `phrase`는 부분일치(`text.count`), `regex`는 `finditer` 횟수.
  - `observe_candidates`(apply:false)는 **감점하지 않되 hit 기록**(메모 관찰용, score 영향 0).
  - 반환에 `hits`(expr·근거·weight·applied 여부) 유지·확장.
- **당일 감점 메모 집계 추가**: `penalty_memo(records) -> dict`
  - 총 감점, 표현별(근거) 횟수·감점합, 매체별 감점합, 관찰후보 hit 수.
- `save_article_report()`에 `penalty_memo` 블록 포함(파일 보존 A안).

### 4.3 스킬 — `.claude/skills/objectivity-score/SKILL.md`(수정)
- 절차 앞에 **fetch 폴백**: `raw/<날짜>.json` 없으면 `python3 src/fetch.py` 먼저 실행.
- 출력에 **당일 감점 메모 + 누적 표** 렌더링 절차 추가.
- observe-only·멱등·커밋 확인 등 기존 주의 유지.

### 4.4 스킬 — `.claude/skills/daily-briefing/SKILL.md`(수정)
- 출력 포맷에 3개 항목 추가(사용자 요청): **①시스템 정상작동 확인 ②실패(무엇·왜) ③우선순위 점수(당일 감점 메모+누적)**.
- ③ 절차에서 `objectivity-score` 스킬을 호출해 결과를 첨부.

## 5. 데이터 흐름(브리핑 시)

```
브리핑 요청
  → 맥락 읽기 + 진행 위치 판단(기존)
  → [신규] 시스템 정상작동 확인(원격 다이제스트·커밋 산출물)
  → [신규] 실패 로그 확인(failures-<날짜>.json 무엇·왜)
  → [신규] objectivity-score 스킬 호출:
        raw 없으면 fetch → objectivity.py <오늘> 채점
        → articles-<오늘>.json(감점분+메모) 저장, media.json 누적 갱신
        → 당일 감점 메모 + 누적 표 반환
  → 브리핑 4섹션 + 위 3항목 출력
  → 시작 확인
```

## 6. 오류 처리

| 상황 | 동작 |
|---|---|
| `raw/<날짜>.json` 없음 | fetch 먼저 실행. fetch도 실패 시 3번 섹션에 "채점 불가(수집 실패)" 표기, 브리핑은 계속 |
| `penalties.yaml` 없음/파싱 실패 | 현행 시드 사전으로 폴백 + 경고 1줄 |
| 이미 처리한 날짜 | 멱등(누적 이중 반영 없음). 메모는 저장된 리포트로 재출력 |
| 정규식 오류 항목 | 로드 시 검증, 문제 항목만 건너뛰고 경고(전체 실패 방지) |

## 7. 테스트(TDD)

- `load_penalties`: 정상 로드 / 파일 없음 폴백 / 잘못된 정규식 스킵.
- `objectivity_score`: weight 합산 / 반복 등장 횟수 감점 / FLOOR 클램프 도달 /
  `observe_candidates` 감점 0·hit 기록 / exclusion 단어 미감점.
- `penalty_memo`: 표현별·매체별 집계 정확성 / 감점 0일 때 빈 메모.
- 회귀: 기존 멱등·active_sources 필터 테스트 유지.
- 실제 데이터 스모크: `raw/2026-07-01.json` 재채점해 매일경제 "반도체가 다 했네" strong 감점 확인.

## 8. 성공 기준

1. `penalties.yaml` 검수본으로 채점 시 판별력이 생긴다(감점 사례가 07-01에서 1건→다수로 증가, 사실 제목은 미감점).
2. 당일 감점 메모가 "총 감점·표현(근거)·매체별"로 렌더링된다.
3. 브리핑 한 번으로 시스템 확인 + 실패 + 당일/누적 점수를 함께 받는다.
4. observe-only 불변: curate/summarize/run.sh/daily.yml 미변경.

## 9. 순서

감점 사전 검수·확정(→ `penalties.yaml`) → 채점 코드 보강(TDD) → 스킬 강화·연결 → 브리핑 통합 검증.
