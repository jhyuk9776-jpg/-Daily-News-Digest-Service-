# 설계: 객관성 점수 변별력 개선 — "중립 매체 그물망"

작성일: 2026-07-06
작성: 이재혁
상태: 설계 확정(사용자 승인) → writing-plans 대기

## 1. 목적

매체 객관성 점수(observe-only)의 **변별력**을 만든다. 현재 `scores/media.json`은
7개 매체가 전부 99.8~100에 몰려 매체를 구분하지 못한다. 채점 로직만 개선해
매체별 점수가 벌어지게 하되 **다이제스트 파이프라인에는 손대지 않는다**(observe-only).

**더 큰 동기(사용자):** AI 시대에 잘못된 정보로 선동하는 매체가 늘어난다. 중립을
유지하고 객관적 사실만 전달하는 매체를 가려내는 "그물망"을 만드는 것이 목표다.
단, 값싼 결정적 규칙이 잡는 것은 **어조·평가·프레이밍(tone/framing)**이며, 이는
**사실의 진위(허위정보 자체)와는 다른 축**이다. 이 망은 "톤·프레이밍으로 편향
의심 매체를 거르는 1차 망"으로 정직하게 포지셔닝하고, 진위 판별은 별도(교차 신호)로 둔다.

## 2. 확정된 방향 (브레인스토밍 결과)

| 항목 | 결정 |
|---|---|
| 모드 | **observe-only 유지**(방향 A). `curate/select/summarize/run.sh/daily.yml` 불변 |
| 집계 지표 | **A안: 감점 밀도** — `1000건당 감점 point`(낮을수록 객관적). 0~100 평균의 구조적 압축을 제거 |
| 채점 텍스트 | 트랙1=제목+리드(전량), 트랙2=제목+리드+**본문**(다이제스트 선별분) |
| 추가 신호 | ① 구조적 낚시, ② 인용·출처 밀도, ③ 어휘 밀도 확장, ④ 교차보도 이상치 (전부 포함) |
| 차등 가중 | 같은 표현이라도 **제목 hit > 본문 hit**(채널별 가중) |
| 가속 감점 | hit가 임계 T를 넘으면 **볼록 가산**(누적 편향 신호), per-기사 상한 |
| 후속(비범위) | 상수 캘리브레이션 + **선별 반영(B안)**은 backfill 분포로 변별력 검증 후 별도 결정 |

**축 분리 원칙:** ②(인용 밀도, 중립 가점)와 ④(교차 이상치)는 감점 point에 **섞지
않고** 별도 관찰축으로 기록한다. 프로젝트가 이미 정한 "가점과 객관성은 다른 축,
증거(숫자) 가점을 객관성에 섞지 말 것"(작업일지 4일차)을 지킨다.

## 3. 범위 / 비범위

- **범위:** `penalties.yaml`(설정 확장) + `src/objectivity.py`(채점·집계 개편) +
  트랙2 본문 감사 스텝 + 스킬 출력 갱신(`objectivity-score`, `daily-briefing`).
- **비범위(후속):** 선별·랭킹 반영(B안), 상수 캘리브레이션 확정, 익명 출처 비율(⑤),
  LLM 실시간 채점, 팩트체크 기반 진위 판별.

## 4. 아키텍처

### 4.1 두 트랙

**트랙 1 · 매체 평판 밀도** (변별력의 핵심)
- 대상: 전량 ~1,000건/일, **제목+리드**, 비용 0(RSS만)
- 감점축(매체 밀도에 반영): 감점 사전(등급 weight) + ① 구조적 낚시 + ③ 어휘 밀도
  확장, 여기에 **가속** 적용
- 지표: 매체 밀도 = `Σ escalated_points / Σ article_count × 1000` — 누적합 비율
  (EWMA α 없음, 표본 누적으로 자연 안정화). **낮을수록 객관적**
- 별도 관찰축(감점 point 미포함): ② 인용·출처 밀도, ④ 교차보도 이상치

**트랙 2 · 다이제스트 본문 감사** (실제 나간 기사 품질)
- 대상: 선별된 N건(설정값: 경제 10 + 나머지 각 2), **제목+리드+본문**
- 본문 hit는 가벼운 가중(`body_factor`). 채널별로 채점 후 합산
- **트랙 1 매체 밀도엔 섞지 않음** — 모집단·텍스트 범위가 달라 오염 방지. 별도 리포트
- 격리: `run.sh`/`summarize`/`curate` 불변. **캐시된 본문**(`cache/`)을 읽는 별도 스텝

### 4.2 감점 사전 — `penalties.yaml`(확장)

- **`scoring:` 블록 신설**(튜너블 상수, 코드 수정 없이 조정. 전부 시드값=데이터 후 보정):
  - `tiers: {strong: 15, medium: 8, weak: 3}`
  - `escalation: {T: 3, step: 5, cap: 45}`
  - `body_factor: 0.5`
- **`scope` 필드**를 penalties 항목에 추가: `title` | `lead` | `body` | `text`(기본).
  구조적 낚시(①)는 `scope: title`.
- **구조적 낚시 항목**(정규식): 추측형 종결(`\?$`, `할까`, `아니냐`, `되나`, `\w+나\?`),
  과도 문장부호(`\?!`, `…`, `!{2,}`) — 제목 한정.
- **어휘 밀도 확장(③)**: 감정·평가 어휘 행 추가(충격/논란/파문/발칵/초강수 등). 오탐은
  `exclusions` + weak tier로 관리.
- **관찰축 설정**: `attribution_markers`(②용 귀속 표지 목록). ④는 코드에서 클러스터로 계산.

### 4.3 채점기 — `src/objectivity.py`(수정)

- **로더 확장**: `scoring` 블록·`scope`·구조 정규식 파싱. 파일 없음/정규식 오류 폴백 유지.
- **per-기사 채점** `objectivity_score()` 개편:
  - 채널별(제목/리드/본문) 매칭 → `scope` 필터 적용 → `body`는 `body_factor` 가중.
  - `raw_points = Σ(tier_weight × count × channel_factor)`.
  - `n_hits` = 감점 매칭 총 횟수. `escalated = min(cap, raw_points + step × max(0, n_hits − T))`.
  - 반환: `{points, hits, n_hits, observe_hits, attribution_count, is_outlier_candidate}`.
- **관찰축**:
  - ② `attribution_count` = 귀속 표지 등장수.
  - ④ `is_outlier_candidate` = (교차검증수==1) & 감점 hit 동반. (클러스터 정보는 curate 재사용)
- **집계** `update_media_scores()` 개편:
  - 매체별 `penalty_points_total += Σ escalated`, `article_count += n`,
    `attribution_total += Σ`, `outlier_total += Σ`.
  - `density_per_1000 = penalty_points_total / article_count × 1000`(파생 계산).
  - 기존 EWMA `score`(0~100)는 **제거/대체**. 스키마가 바뀌므로 `--backfill`로 리셋 재구축.
- **당일 메모** `penalty_memo()`: 밀도·표현별·매체별·관찰축 요약으로 확장.
- **트랙 2**: 선별분(`selected/`) + 캐시 본문을 읽어 본문 포함 채점 →
  `scores/digest-audit-YYYY-MM-DD.json` 저장(트랙1 미오염).

### 4.4 데이터 모델

- `scores/media.json`(스키마 변경): 매체별
  `{penalty_points_total, article_count, density_per_1000, attribution_total, outlier_total, count, last_seen}`.
- `scores/articles-YYYY-MM-DD.json`(확장): 트랙1 당일 감점 메모(밀도·관찰축 포함).
- `scores/digest-audit-YYYY-MM-DD.json`(신규): 트랙2 본문 감사.

### 4.5 스킬 출력(갱신)

- `objectivity-score`, `daily-briefing`: **밀도(낮을수록 좋음) 순위 표** + 관찰축(②④) +
  트랙2 감사 요약으로 렌더링 변경. observe-only·멱등·커밋 확인 유지.

## 5. 데이터 흐름

```
raw/<날짜>.json(전량, 제목+리드)
  → per-기사 채점(사전+구조+어휘, 가속)         → 트랙1 감점 point
  → 관찰축 계산(② 인용밀도, ④ 교차 이상치)      → 별도 기록
  → 매체별 누적(Σpoints/Σcount) → density_per_1000 → media.json
  → 당일 메모 → articles-<날짜>.json

selected/<날짜>(선별 N건) + cache 본문
  → per-기사 채점(제목+리드+본문, body_factor)   → 트랙2 감사
  → digest-audit-<날짜>.json (트랙1 미오염)
```

## 6. 오류 처리

| 상황 | 동작 |
|---|---|
| `penalties.yaml` 없음/파싱 실패 | 시드 사전+기본 `scoring`으로 폴백 + 경고 1줄 |
| 잘못된 정규식 항목 | 로드 시 검증, 해당 항목만 스킵 + 경고 |
| `scope` 미지정 | 기본 `text`(제목+리드) |
| 트랙2 본문 캐시 없음 | 해당 기사 본문 생략(제목+리드로만) + 감사 리포트에 표기, 계속 진행 |
| 스키마 불일치(구 media.json) | `--backfill`로 리셋 재구축(구 `score` 필드 폐기) |
| 이미 처리한 날짜 | 멱등(누적 이중 반영 없음) |

## 7. 테스트 (TDD)

- **로더**: `scoring` 블록 파싱 / `scope` 필드 / 구조 정규식 컴파일 / 파일 없음·정규식 오류 폴백.
- **per-기사 채점**: 등급 weight 합산 / 구조 신호는 제목에만(리드·본문 미적용) /
  어휘 확장 매칭 / 가속(n≤T 선형, n>T 볼록, cap 도달) /
  차등가중(같은 표현 title vs body 다른 점수, body_factor) / observe·exclusions 감점 0.
- **관찰축**: ② 귀속 표지 카운트 정확 / ④ 단독클러스터+hit 동반만 카운트.
- **집계**: `Σpoints/Σcount×1000` 정확 / 누적 멱등 / backfill 리셋 후 **매체별 밀도가 벌어짐**(핵심 스모크).
- **트랙2**: 본문 포함 채점 / 별도 리포트 생성 / 트랙1 밀도 미오염 / 본문 캐시 없음 폴백.
- **회귀**: 기존 `active_sources` 필터·멱등·날짜창 유지.

## 8. 성공 기준

1. `--backfill` 후 media.json 밀도가 매체별로 **유의하게 벌어진다**(현 99.8~100 압축 해소).
2. 사실형 제목은 미감점, 낚시/평가형은 감점 — 정밀도 유지(오탐 스팟체크).
3. observe-only 불변: `curate/select/summarize/run.sh/daily.yml` 미변경.
4. 트랙2 본문 감사 리포트가 생성되고 트랙1 밀도와 분리된다.
5. 관찰축(②④)이 매체별로 축적된다.

## 9. 구현 순서 (phased TDD)

1. `penalties.yaml`: `scoring:` 블록 + `scope` + 구조/어휘 확장 (설정만)
2. `objectivity.py` 로더 확장 (scoring/scope/구조 정규식)
3. per-기사 채점 개편 (등급·구조·가속·차등가중)
4. 관찰축 ②④ 계산·기록
5. 집계 밀도 전환 + media.json 스키마 리셋 backfill ← **변별력 1차 확인 가능**
6. 트랙2 본문 감사 스텝 (캐시 본문 읽기)
7. 스킬 출력(`objectivity-score`/`daily-briefing`) 밀도·관찰축·감사 렌더링 갱신
8. `--backfill` 실행 → 분포 확인(변별력 테스트)

**순서 주의:** 오늘은 main에서 observe-only로 구현·테스트. 상수 캘리브레이션과
"다이제스트 선별 반영(B안)"은 PR #2(feat/extract-quality) 병합 + 신규 매체 데이터
축적 후 별도로 결정한다. 5단계까지만 가도 변별력 여부는 확인된다.
