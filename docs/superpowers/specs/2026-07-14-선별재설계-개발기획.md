### 1.1 지금 어떻게 도는가 (Before)

현재 선별은 [`src/curate.py`](../../../src/curate.py)의 4단계다.

| 단계 | 함수 | 하는 일 |
|---|---|---|

| ② 증거 점수 | `evidence_signals` | 숫자·%·기관·인용·기간 신호 0~5개 카운트 (메타데이터, 선별에 직접 안 씀) |

### 1.2 무엇이 문제인가 (원인)

| # | 원인 | 근거 |
|---|---|---|
| C1 | **대표 선정이 본문 품질과 무관** | `cluster_articles`의 대표는 `priority_map`의 최소값(=수동 우선순위)으로만 뽑힘. 제목만 보고, 본문의 객관성·근거는 안 봄 |
| C2 | **매체 우선순위가 수동** | `sources.yaml`의 `우선순위` 필드를 사람이 관리 → 주관·유지비. 이미 폐지 방향으로 합의됨 |
| C3 | **교차검증수 편향** | 정렬 1순위가 `교차검증수`뿐 → 통신사 물량·받아쓰기 기사가 위로 올라감(물량 = 다양성 아님) |
| C4 | **같은 매체 중복 미처리** | 단일 패스 클러스터링이라 한 매체가 같은 사건을 여러 건 쏟으면 교차검증수가 부풀 수 있음 |
| C5 | **평판 데이터 없음** | 어떤 매체가 반복적으로 대표로 뽑히는지(신뢰도) 축적이 없어 데이터 기반 판단 근거가 없음 |

> 핵심: 지금은 "누가 먼저·많이 썼나"로 대표를 고른다. 바꾼 뒤에는 "누가 **객관적·근거 있게** 썼나"로 고르고, 그 결과를 **데이터로 축적**해 매체 평판을 만든다.

## 2. 목표

| 목표 | 성공의 모습 |
|---|---|
| G1 대표를 **본문 채점**으로 선정 | 클러스터 대표 = `객관성 0.6 + 근거성 0.4` 최고점 기사 |
| G2 매체 평판을 **데이터로 축적** | `media.json`에 매체별 `선택률 = win/appear` 누적 |
| G3 수동 우선순위 **완전 제거** | `sources.yaml 우선순위` 미사용. 매체 서열은 `compute_ranks` density 런타임 파생으로 대체 |

| G5 **유지비 0** | 코어단어·주제·평판 모두 자동 산출. 사람이 손댈 파일 없음 |

**비목표(Non-goals):** LLM 기반 채점 도입(휴리스틱 유지), 카테고리 체계 개편, 웹/이메일 렌더링 변경. 이번 범위는 `curate` 파이프라인과 `scores/` 데이터뿐이다.

## 3. 아키텍처

### 3.1 부품 경계

각 부품은 순수 함수로 독립 테스트 가능해야 한다(입력 dict → 출력 dict, I/O는 경계에서만).

```
raw/<date>.json
   │ (전처리: 날짜 필터 — 블랙리스트는 대표 게이트로 이동, §9 D5)
   ▼
┌───────────────────────────────────────────────────────────┐
│ ① cluster_two_stage   같은 매체 내부 → 매체 간 2단계 클러스터 │
│ ② core_words          코어단어 집계 · 주제 top-3 · 가중치(관찰)│
│ ③ length_filter       본문 300~1500자 (⑥ 게이트로 통합)       │
│ ④ rank_clusters       정렬(등장매체수↓ → 최신순, 2키)         │
│ ⑤ select              최소 3매체만 상위 limit개 (백필 없음)    │
│ ⑥ score+gate          게이트(낚시·블랙·본문관찰) → 대표 채점   │
│ ⑦ update_reputation   appear/win 갱신 · 선택률 → 파이프라인 서열│
└───────────────────────────────────────────────────────────┘
   ▼
selected/<date>.json  +  scores/{core-word-weights,topics-<date>,media,gate-excluded-<date>}.json
```

> ⑤ backfill·⑥ 동점=density·④ 3키 정렬은 이번 사이클에 바뀌었다(§9 D4~D8 반영본이 위 다이어그램).

### 3.2 재사용할 기존 자산 (새로 만들지 말 것)

| 필요 기능 | 기존 자산 | 비고 |
|---|---|---|
| 매체 서열 | `compute_selection_ranks`(선택률) / `scores/media.json` | 파이프라인 서열(§9 D6). density(`compute_ranks`)는 브리핑 관찰축으로만. 재구현 금지 |
| 본문 객관성 감점 | `objectivity.body_objectivity(body, title)` | Phase 0 완료. `scope:title` 격리 적용됨 |
| 본문 근거성 | `objectivity.source_coverage` | Phase 0 완료. 원안의 `body_richness`(근거 밀도)를 대체 — 밀도는 홍보 기사가 더 높게 나오는 함정(§5.6) |

## 5. 상세 설계

### 5.1 ① 2단계 클러스터링

**왜 2단계인가:** C4(같은 매체 중복). 한 매체가 같은 사건을 여러 건 내면 매체 간 교차검증수를 부풀린다. 먼저 매체 내부에서 접고, 그다음 매체 간에 묶는다.

- **1단계 (매체 내부):** 같은 `source` 안에서 제목 유사도 ≥ `SIMILARITY_THRESHOLD`(0.6) 이면 **최근 1건만** 남기고 나머지 축소.
- **2단계 (매체 간):** 남은 대표 후보들을 유사도 **+ 코어단어 공유**로 묶는다. 제목 유사도만으로 부족한 케이스를 코어단어 교집합(≥1개 공유)이 보강한다.

**클러스터 산출물(부품 출력):**
```
{
  "members":  [기사...],        # 매체 간에 묶인 전 멤버
  "sources":  {"연합뉴스", ...}, # 등장 매체 집합
  "media_count": 5,             # = len(sources)  (교차검증수 대체 명칭)
  "core_words": ["관세","상호관세"]
}
```

> 코어단어 공유는 유사도의 보조 신호다. 유사도 단독 오탐(다른 사건인데 제목만 비슷)·미탐(같은 사건인데 표현 다름)을 코어단어 교집합이 완충한다.

### 5.2 ② 코어단어 집계 · 주제 top-3 · 가중치 갱신

**코어단어 추출:** 제목에서 자동 추출(불용어 제거 후 명사 위주). **유지비 0**이 핵심 — 사전 수동 관리 없음.

> **구현 정제(2026-07-17, D2/D3):** "성격 없는 토큰 배제"로 오병합을 막았다. 상세는 §9 참조.
> ① 숫자 토큰 배제(정규식 `[가-힣A-Za-z]{2,}`) — `00`·`20`·`2026` 파편 제거. ② 서술어 blocklist
> 확장(17→52: `개최`·`세미나`·`계약`·`협력` 등 "사건이 일어났다"류). ③ `tokenize()` 토큰 단위 매칭
> — `하나`가 `하나은행` 안에서 잡히던 부분문자열 오탐 제거. ④ 코어단어 가산 상한 `0.15`(1개어치) —
> 서술어 2개만 겹쳐도 유사도 0을 병합으로 밀어올리던 것 차단(코어단어는 유사도의 보조 신호).

**주제 점수:**
```
topic_score = 등장매체수 + 총기사수 × 0.2
```
- 매체수를 주가중으로 둔 이유: **다양성이 물량보다 객관적** 신호(통신사 1곳이 10건 쏜 것 < 서로 다른 5개 매체가 1건씩).
- 상위 3개 주제를 그날의 top-3로 확정 → `topics-<date>.json` 저장.

**가중치 갱신 (누적, `core-word-weights.json`):**
- 순위 가점: 1위 `5`, 2위 `3`, 3위 `1`
- 학습률 `α = 0.1`
- 갱신식: `weight[word] += rank_score × α` (top-3 주제에 속한 코어단어 대상)
- **빈 저장소 첫날:** top-3의 가점값이 그대로 초기값이 된다.

> ⚠️ 결정 필요: 감쇠(decay) 없이 가산만 하면 장기적으로 가중치가 단조 증가한다. 초기 운영에서는 가산만으로 두고, 값이 포화하면 EMA(`weight = weight×(1-α) + rank_score×α`) 또는 일정 주기 재정규화를 도입한다. **Phase 2 구현 시 실측 후 결정.**

### 5.3 ③ 길이 필터 (300~1500자)

**규칙:** 본문 글자수 `300 ≤ len ≤ 1500` 밖이면 대표 후보에서 이탈.

**지연 적용 (중요):** 논리적으로 3단계지만 **실행은 대표 채점(⑥) 직전**이다. 이유 = 전 후보의 본문을 미리 추출하면 비용이 폭증한다(C-비용). 따라서 정렬(④)로 상위 후보를 좁힌 뒤, **살아남은 클러스터의 대표 후보 본문만 추출**하며 길이 필터를 건다. 길이 필터는 대표 게이트(⑥)의 하드 게이트 중 하나로 통합됐다(§5.6).

- 이탈 시 **로그 남김**(침묵 금지): 어떤 기사가 왜(너무 짧음/너무 김) 탈락했는지 기록.
- 길이 필터로 대표 후보가 전부 탈락하면 → 그 클러스터의 차선 멤버로 재시도, 그래도 없으면 클러스터 드롭(로그).

### 5.4 ④ 정렬 (구현 확정: D8, 2026-07-17)

클러스터 정렬키(위로 올라올수록 우선) — **2키로 단순화**:

| 순위 | 키 | 방향 | 의미 |
|---|---|---|---|
| 1 | `corroboration_count`(등장 매체 수) | 내림차순 | 여러 매체가 다룬 사건 우선(다양성) |
| 2 | 발행 최신순 | 내림차순 | 동수 매체 동점 tie-break |

> **원안(코어단어 포함여부·가중치합 키)을 뺀 이유:** 코어단어 가중치는 현재 **관찰·기록용**으로만 쓴다
> (데이터 축적 후 정렬 반영 재검토). 선별 축을 "매체 다양성 = 사건의 사실성" 단일로 두는 게 서비스
> 목적(판단 최소화)에 맞다. 상세는 §9 D8.

### 5.5 ⑤ 선정 — 최소 3매체, 백필 없음 (구현 확정: D8, 2026-07-17)

- **최소 3매체**(`MIN_MEDIA = 3`) 통과분만 대표 후보. 미만(단독·2매체)은 드롭 — **억지로 안 채운다**(로드맵 §8).
- **분야별 상한**(`limits.yaml`, 기본 2·경제 10)만큼 상위에서 선정. 상한 미달이어도 그대로 둔다.

> **⚠️ 원안의 density 라운드로빈 백필은 삭제됐다.** 백필은 원래 "정렬 상위가 소수 매체로 채워지는 것"을
> 막으려 단독 클러스터로 빈 슬롯을 메우는 장치였다. 그런데 낚시·홍보 물량이 그 빈 슬롯으로 딸려 들어왔다.
> 이제 낚시·홍보는 **대표 게이트(§5.6, D4/D5)**가 기사 단위로 걸러내므로, 백필의 존재 이유가 사라졌다.
> `backfill_round_robin` 함수 삭제. 상세는 §9 D8.

### 5.6 ⑥ 대표 채점표 + 통합 게이트 (게이트: D4/D5, 2026-07-17)

살아남은 클러스터별로, 멤버 본문을 추출하며 **하드 게이트 → 채점 최고점**으로 대표를 선정한다.
게이트는 본문을 이미 추출하는 이 지점에 통합해 추가 비용 0이다.

**하드 게이트(대표 자격 박탈, 멤버로는 남아 교차검증엔 기여):**

| 게이트 | 판정 | 근거 |
|---|---|---|
| 길이 | 본문 300~1500자 밖 | §5.3 |
| 블랙리스트 | `reporters.reporter_key ∈ blacklist` | 부실 기자 기사도 사건 증거로는 유효 → 멤버로만 남김(D5) |
| 제목 낚시 | `objectivity.title_penalty(title) > 0`(scope:title 물음표·추측형 종결 등) | 편파적 제목은 대표로 노출 안 함(D4) |

**관찰 게이트(로그만, 배제 안 함):** 본문 `total < REP_SCORE_FLOOR(0.35)`. 라벨 2개뿐이라 임의 하한으로
진짜 기사를 떨구는 대신 분포를 며칠 관찰한 뒤 하드 하한으로 승격 예정(D4 관찰 모드).

**채점(게이트 통과 멤버 중 최고점 = 대표):**
```
score = 0.6 × 객관성 + 0.4 × 근거성
```

| 성분 | 계산 | 근거 |
|---|---|---|
| 객관성 (0~1) | `objectivity.body_objectivity(body,title)`의 감점을 `1 - min(감점, 16)/16`로 역정규화. 감점↑ → 점수↓ | 서비스 목적이 객관성 → 주가중 0.6 |
| 근거성 (0~1) | **독립 출처 커버리지** = (독립 출처를 명시한 문장 수) / (전체 문장 수). `objectivity.source_coverage` | 홍보 기사의 자기인용을 근거로 안 침 |
| 동점 처리 | **선택률 순위**(`compute_selection_ranks`) → **최신순** | D6/D7. density(연좌제)를 뺀 파생 서열 → 첫날처럼 서열 무의미하면 최신순 |

**왜 근거성 = 독립 출처 커버리지인가 (구현 중 변경, 2026-07-14):** 원안은 "증거 신호(숫자·%·기관) ≥1개인
문장 비율"인 **문장 커버리지**였다. 총량 대신 비율을 쓴 이유는 유효했지만(감점1호 실측: 홍보인데 근거밀도
3.59 = 길이로 이기는 **함정**), 구현하며 **비율로도 홍보를 못 거른다**는 걸 발견했다. 보도자료는 자사 수치를
문장마다 뿌려서 커버리지까지 높게 나온다. 그래서 "숫자가 있나"가 아니라 **"누가 밝혔나"**로 기준을 옮겼다:
독립기관(통계청 등) 인용은 인정, 귀속표지(`~에 따르면`)는 자기지칭어가 없을 때만 인정. 이러면 자기인용으로
도배한 홍보 기사의 근거성이 0으로 떨어진다.

> **역정규화 상수 확정(A2 해소):** `PENALTY_CAP` → `objectivity.OBJ_PENALTY_FULL = 16`. 원안 출발값은
> 감점1호(홍보) 실측 8이었으나, 8을 하한으로 잡으면 medium 위반 1건짜리 기사가 객관성 0이 되어 과하게
> 가혹했다. 16 = medium 2건을 하한으로 두면 medium 1건이 정확히 0.5가 되어 눈금이 자연스럽다.
> 라벨 2개로 잡은 시드값이므로 데이터 축적 후 tier 가중치와 함께 재보정한다(코드에 `ponytail:` 주석으로 표시).

### 5.7 ⑦ 요약 + 평판 갱신

- 최종 선정 기사로 요약 생성(기존 요약 파이프라인 재사용).
- **평판 갱신 (`media.json`):**
  - 클러스터 **멤버**로 등장한 매체: `appear += 1`
  - 클러스터 **대표**로 뽑힌 매체: `win += 1`
  - `선택률 = win / appear`
- **왜 원시 횟수가 아니라 선택률인가:** 통신사는 물량이 많아 `win` 절대값이 커진다. 정규화(선택률)해야 "나온 만큼 대표로 뽑히는 비율"이라는 공정한 신뢰도가 된다(C3와 같은 편향 방지).

> **선택률이 파이프라인 서열이 됐다(D6, 2026-07-17):** 선택률은 이제 단순 기록이 아니라 **매체 서열**로
> 쓰인다(`compute_selection_ranks` → 대표 동점 tie-break). 원래 서열이던 density는 매체 단위 집계라
> 연좌제(좋은 매체의 낚시 기사는 통과, 나쁜 매체의 좋은 기사는 밀림) 문제가 있어 **파이프라인에서 빼고
> 브리핑 관찰축(`compute_ranks`·rank-history)으로만 유지**한다. 낚시·홍보 제거라는 density의 원래 의도는
> 대표 게이트(§5.6)가 기사 단위로 대신한다. 첫날처럼 선택률이 전부 0이면 최신순(D7)이 tie-break을 이어받는다.

## 6. 데이터 스키마

모든 파일은 `scores/` 아래. UTF-8, `indent=2`, `ensure_ascii=False`.

### 6.1 `scores/core-word-weights.json` (누적, 단일 파일)

```json
{
  "updated": "2026-07-14",
  "weights": {
    "관세": 1.8,
    "금리": 1.2,
    "반도체": 0.9
  }
}
```
- `weights`: 코어단어 → 누적 가중치(float). ②에서 매일 갱신.
- 파일 없으면 `{"updated": null, "weights": {}}`로 폴백(첫날 = 초기값 기록).

### 6.2 `scores/topics-<date>.json` (일별, 날짜당 1개)

```json
{
  "date": "2026-07-14",
  "topics": [
    {"rank": 1, "core_words": ["관세", "상호관세"], "media_count": 7, "article_count": 12, "score": 9.4},
    {"rank": 2, "core_words": ["금리", "기준금리"], "media_count": 5, "article_count": 6,  "score": 6.2},
    {"rank": 3, "core_words": ["반도체"],           "media_count": 4, "article_count": 4,  "score": 4.8}
  ]
}
```
- `score = media_count + article_count × 0.2`
- top-3만 기록. 감사·재현용(그날 왜 이 주제가 top이었는지).

### 6.3 `scores/media.json` (기존 파일에 필드 추가)

기존 density 필드는 그대로 두고 평판 3필드 추가:

```json
{
  "연합뉴스": {
    "…기존 density 필드…": "그대로 유지",
    "appear": 120,
    "win": 18,
    "selection_rate": 0.15
  }
}
```
- `appear`/`win`: 정수 누적. `selection_rate = win / appear` (appear=0이면 `null` 또는 0).
- **하위호환:** 기존 매체 엔트리에 3필드 없으면 최초 갱신 시 0에서 시작. `compute_ranks`는 이 필드를 무시하므로 density 파생에 영향 없음.

- 버린 것: 없음(순수 추가). 그럼에도: Phase 6 반복 검증의 안전망.

**D4·D5 통합 게이트** (`4d5d734`)
- 문제: 대표 채점이 `max`만 하고 하한이 없어 3매체 문턱 넘은 보도자료·낚시가 대표로 노출.
- 해결: 대표 선정 시점에 하드 게이트(제목 낚시·블랙리스트·길이) + 관찰 게이트(본문 점수) 통합. 블랙리스트를 pool 선제거→게이트로 이동.
- 버린 것: 블랙리스트 선제거가 주던 pool 축소(교차검증 카운트가 약간 커짐).
- 그럼에도: 부실 기자 기사도 사건 증거로는 유효 → 멤버로 남기고 대표만 제외. 낚시·홍보는 매체 단위(density 연좌제)가 아니라 기사 단위 판정이 의도에 맞다.

**D6·D7 선택률 서열 + 최신순 tie-break** (`c7ad5be`)
- 문제: 매체 서열이 density(감점 밀도)였는데, 매체 단위 집계라 연좌제(좋은 매체의 낚시 통과, 나쁜 매체의 좋은 기사 밀림).
- 해결: 파이프라인 서열을 선택률(win/appear)로 교체(`compute_selection_ranks`). density는 브리핑 관찰축으로만. 동점 tie-break에 최신순 추가.
- 버린 것: density가 주던 누적 객관성 정보(cold-start에 유용).
- 그럼에도: density의 원래 의도(낚시·홍보 제거)는 대표 게이트가 대신하고, 선택률이 더 직접적인 "뽑히는 비율" 신뢰도. 첫날 공백은 최신순이 완충.

**D8 최소 3매체 + 정렬 단순화 + 백필 삭제** (`4ecb46e`)
- 문제: 정렬에 코어단어 가중치가 섞이고, 단독 백필이 빈 슬롯을 낚시·홍보 물량으로 채웠다.
- 해결: 최소 3매체만, 매체 다양성↓→최신 2키 정렬, 백필 함수 삭제. 코어단어 가중치는 관찰용으로 강등.
- 버린 것: 백필이 채우던 볼륨(3매체 미달 사건 노출 안 됨 — 오늘 실측 하루 ~28~31건).
- 그럼에도: 백필의 존재 이유(물량 보정)를 대표 게이트가 대체. "억지로 안 채운다"는 로드맵 §8 원칙과 일치.
