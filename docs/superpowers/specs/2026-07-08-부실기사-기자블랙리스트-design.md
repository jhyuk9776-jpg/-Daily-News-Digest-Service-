# 부실 기사 기자 블랙리스트 — 설계

작성: 2026-07-08 / 이재혁 · Claude
성격: 본문이 없거나 부실한 기사를 반복 생산하는 기자를 감지·누적하고, 일정 점수에 도달하면
그 기자의 기사를 선별에서 제외하는 기능의 설계 문서.

관련: 품질이슈 로그(ISSUE 계열)와 결이 같은 "부실을 자산으로 축적" 방침의 연장.
객관성 스코어러([04-객관성-점수축적기](../../../기획/시스템기획/기능설계/04-객관성-점수축적기.md))가
**매체** 평판을 다룬다면, 이 기능은 **기자** 단위 부실 이력을 다룬다.

---

## 1. 목적

본문이 아예 없거나 한두 문장뿐인 부실 기사를 반복해 내보내는 기자를 감지·누적하고,
누적 점수가 3점에 도달하면 해당 기자의 기사를 **선별 단계에서 제외**한다.

## 2. 점수 규칙 (대표 기사 본문 판정 기준)

매일 다이제스트에 실리는 **대표 기사**(선별·요약 대상)의 본문만 판정한다.
판정은 기존 `extract.extract_body(대표 link, title)` 결과를 재사용한다.

| 판정 | 조건 | 처리 |
|---|---|---|
| **본문 없음** | `extract_body` → None (추출 실패 = `MIN_BODY` 80자 미만 포함) | **즉시 +1점** |
| **부실** | 본문 길이 80자 이상 ~ 200자 미만 | **sparse_count +1** — 3 도달 시 +1점, sparse_count를 0으로 리셋 |
| **정상** | 본문 길이 ≥ 200자 | 변화 없음 |

- **blacklist 조건:** `points >= 3 → blacklisted = true`.
- 도달 경로 예: 본문없음 3회(3점) / 부실 9회(3카운트×3=3점) / 혼합.
- 200자 기준 근거: 한국어 기사 한 문장이 대략 40~70자 → 200자 미만 ≈ 2~3문장 이하를 "부실"로 본다.

## 3. 키 & 정규화

- **키:** `"{매체}::{기자명}"` (예: `"연합뉴스::황철환"`).
  - 같은 이름이라도 매체가 다르면 다른 사람 → 매체로 스코프.
  - 대표 기사의 `author`를 사용(다이제스트에 실제로 실리는 기사).
- **기자명 정규화:** 끝의 "기자" 접미사 제거 + 앞뒤 공백 trim.
  - 예: `"김송이 기자"` → `"김송이"`, `"황철환"` → `"황철환"`.
  - 복수 저자·비정형 표기는 드물어 문자열 전체를 정규화해 그대로 키로 쓴다(과도한 파싱 지양, YAGNI).

## 4. 저장

- **위치:** `scores/reporters.json` (매체 평판 `scores/media.json`과 위치 관례 일치).
- **구조:**

```json
{
  "연합뉴스::황철환": {
    "sparse_count": 0,
    "points": 0,
    "blacklisted": false,
    "last_updated": "2026-07-08",
    "history": [
      { "date": "2026-07-08", "link": "https://...", "reason": "empty", "chars": 0 },
      { "date": "2026-07-08", "link": "https://...", "reason": "sparse", "chars": 123 }
    ]
  }
}
```

- `history`는 오탐 추적용(왜 감점됐는지). 객관성 스코어러가 history로 `…` 오탐(ISSUE-003)을
  잡아낸 것과 같은 이유로 유지한다.
- **중복 기록 방지:** 같은 `(date, link)`가 history에 이미 있으면 재기록하지 않는다
  (같은 기사가 재실행돼도 이중 감점 금지).

## 5. 파이프라인 변경 지점

### 5.1 `src/reporters.py` (신규 — 공유 모듈)

`curate`와 `summarize`가 공유하는 얇은 부품. reporters.json I/O를 한곳에 모은다.

- `normalize_author(name: str) -> str` — "기자" 접미사·공백 제거.
- `reporter_key(source: str, author: str) -> str` — `"{source}::{정규화}"`.
- `load() -> dict` / `save(data: dict)` — 파일 없으면 `{}`.
- `blacklisted_keys(data: dict) -> set[str]` — `blacklisted=true`인 키 집합.
- `record_strike(data, source, author, date, link, reason, chars)` — 점수 규칙(§2) 적용.
  `reason`은 `"empty"` | `"sparse"`. `(date, link)` 중복이면 no-op. 갱신된 dict 반환.

### 5.2 `src/fetch.py`

- RSS 파싱 시 `entry.author`(feedparser 제공)를 raw 기사 dict에 **`author` 필드로 저장**.
  - 값이 없으면 빈 문자열. 정규화는 하지 않고 **원문 그대로** 저장(raw는 원자료 보존).
- raw는 전량 온전 유지 → 객관성 스코어러 관찰 대상이 줄지 않는다.

### 5.3 `src/curate.py`

- 선별 시작 시 `reporters.load()` → `blacklisted_keys()`로 제외 집합 생성.
- 클러스터링 **전에**, 후보 기사 중 `reporter_key(art.source, art.author)`가 제외 집합에 있으면 제거.
- `author` 필드가 없는 옛 raw는 키가 안 만들어져 **자동 통과**(아무도 막지 않음, 안전 우선).
- raw 파일 자체는 건드리지 않는다(필터는 메모리 상 후보 풀에만 적용).

### 5.4 `src/summarize.py`

- 각 선별 항목의 **대표 기사** 본문을 판정해 `reporters.record_strike()` 호출 → `reporters.save()`.
- 판정 재료: 대표 기사 링크의 `extract_body` 결과.
  - **이중 추출 금지:** 대표 본문을 한 번 추출해 판정과 요약 입력에 함께 쓰도록 요약 흐름과 통합한다
    (현재 `iter_contents` 제너레이터 구조와의 배선은 구현 계획에서 확정).
  - None → `reason="empty", chars=0`; 80~200자 미만 → `reason="sparse"`; ≥200자 → 기록 안 함.
- 대표 기사 `author`가 비어 있으면(RSS 미제공/옛 raw) 판정·기록 **스킵**.

## 6. 정책 판단 (확정)

- **즉시 적용(관찰전용 단계 없음).** 사용자 스펙("3번 그러면 제외")대로 바로 제외하되,
  blacklist 도달에 3점 완충이 있어 단발 오탐으로는 제외되지 않는다.
- 오탐이 쌓이면 `history`로 사후 검증·수동 해제(파일에서 `blacklisted`/`points` 직접 수정)한다.

## 7. 엣지 케이스

| 상황 | 처리 |
|---|---|
| `author` 미제공(RSS 누락) | 판정·필터 모두 스킵 |
| 옛 raw(author 필드 없음) | curate 필터 통과(안전) |
| 같은 기자 하루 여러 기사 | 대표로 실린 기사마다 판정. `(date, link)` 중복만 방지 |
| 같은 날 재실행 | history 중복 방지로 이중 감점 없음 |
| 이미 blacklist된 기자 | curate에서 후보 제거되어 더 이상 판정 대상 아님(점수 고정) |

## 8. 테스트

- **`tests/test_reporters.py`**: 정규화("김송이 기자"→"김송이"), 점수 로직
  (본문없음 +1점, 부실 3회=+1점+리셋, points 3→blacklisted), `(date,link)` 중복 방지.
- **`tests/test_curate_limits.py`(또는 신규)**: 블랙리스트 `(매체,기자)` 기사가 후보에서 제외되는지,
  author 없는 기사는 통과하는지.
- **`tests/test_collect_digest.py`(fetch)**: `entry.author`가 raw `author` 필드로 저장되는지(픽스처).

## 9. 범위 밖 (YAGNI)

- 부실 이외 사유(사실오류·낚시 등)로 기자 감점 — 이 사이클 범위 아님.
- 기자 자동 복권(시간 경과 회복) — 필요 시 수동 해제로 충분. 후속 후보.
- 웹 UI 노출 — 무관.
