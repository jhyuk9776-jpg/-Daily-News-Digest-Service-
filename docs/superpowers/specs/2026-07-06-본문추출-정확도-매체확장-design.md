# 본문 추출 정확도 개선 + 매체 확장 — 설계

작성: 2026-07-06 / 이재혁 · Claude
성격: 다이제스트 품질 결함(ISSUE-001 제목-내용 불일치, ISSUE-002 과요약·수치 누락)의 근본 원인인
**본문 추출 오류**를 고치고, 선별 견고성을 위해 **매체를 확장**하는 설계.
관련: `docs/품질이슈-로그.md`(ISSUE-001·002), `기획/decision-log.md`, `기획/일정/일정-로드맵.md`.

## 1. 배경 — 근본 원인 (실측 확인)

`extract.extract_body`가 **기사 본문이 아니라 페이지 하단 "추천/관련 기사 목록"의 `<p>`들을 긁는다.**
2026-07-06 실측:
- 한국경제 식량가격 기사 → 추출 결과가 "K푸드 중동수출", "양곡법" 등 **다른 기사 미리보기**(본문의 하락률 미포함)
- 한국경제 올영 기사 → 추출 결과가 "외국인 쇼핑성지", "택스리펀드" 등 → 다이제스트의 잡탕 불릿과 일치

`soup.find("article")` + 모든 `<p>` 방식이 진짜 본문 컨테이너를 못 찾고 추천 위젯을 포함해서다.
이 하나의 버그가 두 이슈를 모두 설명한다:
- **ISSUE-001:** 본문 대신 남 기사 목록을 요약 → 제목과 무관한 잡탕
- **ISSUE-002:** 본문(숫자)에 도달 못 함 + RSS 요약이 40자만 넘으면 본문 추출을 건너뜀

## 2. 결정 사항 (2026-07-06)

| 항목 | 결정 |
|---|---|
| 추출 방식 | **A안 — 도메인별 선택자 + 검증 가드.** readability 라이브러리는 매체 대폭 확장 시 재검토 |
| 점수 민감도 튜닝 | **이번 범위 밖(별도 후속 브레인스토밍).** 매체 확장으로 데이터가 바뀐 뒤 진행 |
| 서비스 성격 | **지금은 "개인용" 제작 우선.** 무료 공개·법무 라이트 체크는 **유료화 붙일 때 함께**로 연기 |
| 매체 확장 | 개인·비상업 RSS 이용 범위. 신규 6개 매체 추가(§5) |

> 로드맵 영향: "무료 웹 공개 07-10" → "개인용 우선"으로 방향 조정. 마무리 시 decision-log·로드맵에 반영(§8).

## 3. 본문 추출 재작성 — `extract.py`

### 3.1 도메인별 선택자 맵 + 폴백
- `SITE_SELECTORS: dict[domain, list[css_selector]]` — 매체 도메인 → 진짜 본문 컨테이너 선택자.
  우리 소스(기존 7 + 신규 6) 도메인마다 지정. 값은 구현 시 실제 페이지로 확인해 채운다.
- `extract_body(url)`:
  1. 도메인 판별 → 선택자가 있으면 그 컨테이너 내부 `<p>`만 수집
  2. 선택자가 없거나 결과가 비면 기존 휴리스틱(`article`/전체 `<p>`)로 폴백
  3. 결과에 **검증 가드**(§3.2) 적용. 통과 못 하면 `None`

### 3.2 검증 가드 (핵심 — 쓰레기 요약 차단)
추출 텍스트가 진짜 본문인지 확인한다. 실패 시 `None`(→ 상위에서 `extract_failed`).
- **보일러플레이트/추천위젯 탐지:** 다음 신호가 임계 이상이면 기각
  - "ADVERTISEMENT", "무단전재", "재배포 금지" 등 상투 문구
  - 날짜 패턴(`YYYY.MM.DD`)이 3개 이상(추천 목록은 항목마다 날짜) → 본문 아님 신호
- **제목-본문 관련성:** 제목에서 뽑은 핵심 명사(길이 2+ 토큰) 중 **최소 1개 이상이 본문에 등장**해야 통과.
  → 제목과 무관한 본문(다른 기사)을 원천 차단(ISSUE-001의 구조적 방지). 폴백으로 넘어간 관련 매체 본문도
  이 검사를 통과해야만 쓰인다.

### 3.3 RSS 단축 문제 수정 — `iter_contents` (ISSUE-002)
- 현재: RSS 요약 ≥40자면 그걸 먼저 yield → 본문(숫자) 미도달.
- 변경: **대표 매체는 검증 통과한 본문을 우선 yield**, 본문 추출이 실패(None)할 때만 RSS 요약을 yield.
  → 하락률·금액 등 수치가 본문에서 확보된다. 관련 매체 폴백도 동일(본문 우선, 검증 통과 조건).

## 4. 요약 프롬프트 보강 — `summarize.SYSTEM_PROMPT`
- 규칙 추가: "**숫자·비율·금액·기간·기관명은 반드시 보존**한다. 원문에 수치가 있으면 요약에 포함한다."
- 증거 우선(결정로그 1.1) 강화. 프롬프트만 수정(형식 규칙은 유지).

## 5. 매체 확장 — `sources.yaml` (개인용, RSS는 구현 시 검증)

| 신규 매체 | 분야 | 이유 | 비고 |
|---|---|---|---|
| **연합뉴스** | 경제·사회·세계·IT | 통신사 wire copy = **객관성 앵커 + 교차검증** 최강. 도메인 1개로 전 분야 | RSS 미제공 시 **뉴시스/뉴스1**로 대체 |
| **KBS** | 경제·사회·세계·IT | 공영·보도전문, 속보·사실 위주 | |
| **YTN** | 사회·세계 | 24시간 보도, 속보·사실 위주 | |
| **MBC** | 사회·세계 | 공영·지상파 보도, 속보·사실 위주 | |
| **한겨레** | 경제·사회·세계 | 성향 다양성 → 향후 변별력(감점사전 stress-test) | |
| **중앙일보** | 경제·사회·세계 | 반대 성향 축 → 점수 분산 균형 | |
| **블로터** | IT/테크 | 테크 전문 보완 | |

- 각 매체 **RSS URL·분야 매핑·우선순위**는 구현 시 확정. **연합뉴스는 객관성 앵커로 분야별 상위 우선순위**에 둔다
  (대표 기사로 뽑혀 본문 요약 대상이 되도록). 성향지(한겨레·중앙)는 하위 우선순위.
- 신규 7개 도메인 각각 **본문 선택자(§3.1) 추가** 필요. 선별 후보가 늘어 더 객관적인 기사를 고를 폭이 커진다
  (최종 16건 상한은 유지).

## 6. 테스트 — `tests/` (unittest + 저장 HTML 픽스처)

네트워크 없이 검증하기 위해 실제 페이지 HTML 조각을 픽스처로 저장해 사용한다.
- `test_extract.py`(신규):
  - **진짜 본문 추출:** 한국경제 픽스처 → 본문 텍스트 추출, 추천위젯 텍스트 미포함
  - **수치 보존:** 수치가 든 본문 픽스처 → 추출 결과에 숫자 포함
  - **추천위젯 기각:** 추천목록 위주 HTML → `extract_body` None
  - **제목-본문 불일치 기각:** 제목과 무관한 본문 → None
  - **RSS 폴백:** 본문 추출 실패 시 RSS 요약이 쓰이는지(`iter_contents` 순서)
- 기존 `test_fallback.py`(폴백 체인)와 정합 유지.

## 7. 아키텍처 · 데이터 흐름 (변경 요약)

```
fetch(RSS, 신규매체 포함) → curate(선별·중복제거)
  → summarize.iter_contents  ← extract.extract_body(도메인 선택자+검증)  ← [변경]
      · 대표 매체 본문 우선(검증 통과) → 없으면 RSS 요약 → 없으면 관련 매체 본문
  → summarize_one(SYSTEM_PROMPT: 수치 보존)  ← [변경]
  → News/<date>.md
```
- 격리: 추출 정확도는 `extract.py`, 후보 순서는 `iter_contents`, 요약 규칙은 프롬프트로 각각 분리 → 독립 테스트.

## 9. 범위 밖 (YAGNI)

- 객관성 점수 민감도(임계값·증감폭) 튜닝 — 별도 후속
- readability 라이브러리 도입 — 매체 대폭 확장 시
- 법무 라이트 체크 — 유료화 시
- 새 매체 RSS 자동 발굴 — 수동 유지
