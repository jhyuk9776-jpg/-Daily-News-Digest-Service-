# 설계: 감점메모 제거 + 분야별 선택률 + 기자 선택 기록

작성일: 2026-07-23
관련: 선별재설계(feat/selection-redesign, 14일차 라이브), density 완전 제거(9503050)

## 배경

선별재설계가 프로덕션 cron에 라이브된 뒤 첫 관찰일(2026-07-23). 브리핑 §0의
관찰 지표를 선별재설계와 정합하게 정리한다. 세 가지 변경:

1. **감점메모/매체별 감점 제거** — 측정 지점이 선별재설계와 다름
2. **분야별 선택률 추가**(전체와 별도) — 매체의 분야별 강점 파악
3. **기자 선택 기록 추가** — 최종 선택 기사의 기자 관찰

## 문제 정의

### 1. 감점메모는 선별재설계와 무관한 지표다

당일 감점메모·매체별 감점(`by_source`)은 `objectivity.process_date` →
`dated_articles_for`가 만든다. 대상은 **당일창 + 활성매체 필터만 건 수집 원본
전체**(제목+리드)이며, 2차 클러스터링도 하드게이트도 거치지 않는다.

반면 선별재설계가 다루는 **선택률(win/appear)**은 게이트·클러스터 **이후** 대표
승률이다. 둘은 파이프라인의 다른 지점을 측정한다:

| 지표 | 측정 지점 | 대상 | 채점 범위 |
|---|---|---|---|
| 당일 감점메모 / by_source | 클러스터·게이트 **이전** | 수집 원본 전체 | 제목+리드 |
| 선택률 (win/appear) | 클러스터·게이트 **이후** | 교차검증 클러스터 멤버·대표 | 대표 선정 |
| 본문 감사 (digest-audit) | 최종 선택 후 | 다이제스트 대표 | 본문 |

감점메모는 유입 단계 관찰이라 선별재설계 후속 관찰에서 노이즈다 → 제거.

### 2. 선택률이 매체 전역 하나뿐이라 분야 강점이 안 보인다

현재 `selection_rate = win/appear`는 매체 단위 전역값. 한 매체가 IT/테크엔
강하고 세계엔 약한 식의 분야별 강점이 드러나지 않는다. 분야는 4개(경제·사회·
세계·IT/테크)이며, `selection_stats`는 이미 분야 루프 안에서 생성되므로
`category`가 그 자리에 있다([curate.py] `select`).

**소표본 보정은 하지 않는다.** 적은 후보로 모두 선택된 매체(예: 2/2=1.0)가
높은 점수인 것은 의도된 동작이다. 생짜 `win/appear`를 유지한다.

### 3. 최종 선택 기사의 기자가 기록되지 않는다

`reporters.py`가 이미 기자 단위 **부실 스트라이크**(blacklist)를 누적하지만,
"선택됨" 카운터는 없다. 관찰용으로 최종 대표 기사의 (매체, 기자, 선택 횟수)를
기록한다.

## 비목표 (Non-goals)

- 선택률 소표본 보정(베이지안 수축 등) — 하지 않음, 생짜 비율 유지
- 기자 선택 카운트를 선별 가중치·블랙리스트에 반영 — 순수 관찰만
- `score_article`·본문 점수·낚시 게이트 로직 변경 — 불변(공유 코드)

## 설계

### A. 감점메모 제거

**objectivity.py에서 제거(감점메모 전용):**
- `process_date`, `penalty_memo`, `save_article_report`, `dated_articles_for`,
  `outlier_flags`, `attribution_count`, `active_sources`, `run_backfill`
- `main()`의 리포트 생성·`--backfill` CLI 부분
- 결과물 `scores/articles-<date>.json` 산출 중단

**유지(공유 코드 — 낚시 게이트·본문 점수·digest_audit·tie-break이 사용):**
- `score_article`, `body_objectivity`, `representative_score`, `title_penalty`,
  `source_coverage`, `compute_selection_ranks`, `update_selection_rates`,
  `update_rank_history`(정의), `load_store`/`save_store`, `in_date_window`(import)

**이전:** `update_rank_history(date, compute_selection_ranks(store))` 호출을
objectivity.main()에서 curate로 옮긴다. 위치는 `save_store(store)` 직후
(non-dry-run 블록). curate가 이미 store를 로드·갱신·저장하므로 자연스럽다.

**cron:** `.github/workflows/daily.yml`의 "객관성 점수 채점(observe-only)"
단계(`python3 src/objectivity.py`) 삭제. rank_history가 run.sh(curate)에서
갱신되므로 별도 단계가 불필요하다. 커밋 스텝의 `git add scores/`는 유지
(media.json·reporters.json·gate-excluded 계속 커밋).

**스킬:** `.claude/skills/objectivity-score/` 은퇴(디렉터리 삭제). 이 스킬의
유일한 역할이 감점메모 러너였다.

**정리:** 과거 `scores/articles-*.json` 전부 삭제.

### B. 분야별 선택률 (전체와 별도)

**curate.py — selection_stats에 category 추가:**
```python
selection_stats.append({
    "members": [m["source"] for m in c["members"]],
    "winner": rep["source"],
    "category": category,          # 추가
})
```

**objectivity.update_selection_rates — by_category 중첩 누적:**
`media[src]`에 기존 전역 `win_total`/`appear_total`/`selection_rate`를 유지하고,
추가로 `by_category`를 둔다:
```
media[src] = {
    "win_total": int, "appear_total": int, "selection_rate": float|None,
    "by_category": {
        "경제": {"win_total": int, "appear_total": int, "selection_rate": float|None},
        ...
    }
}
```
각 클러스터 누적 시 전역과 해당 분야를 함께 증가시킨다. `selection_rate =
win/appear`(등장 0이면 None), 소표본 보정 없음. 날짜 멱등(`selection_dates`) 유지.

**리셋:** `media.json`을 `{"media": {}, "selection_dates": []}`로 리셋(구조
변경, 07-22 리셋 선례대로).

### C. 기자 선택 기록 (reporters.py 확장, 신규 파일 없음)

**reporters.py — record_selection 추가:**
```python
def record_selection(data, source, author, date, link):
    """최종 대표로 선택된 기사의 기자에 선택 카운트를 더한다. (date,link) 멱등."""
    key = reporter_key(source, author)
    rec = data.setdefault(key, {... 기존 기본값 ..., "selected_count": 0})
    rec.setdefault("selected_count", 0)   # 기존 레코드 호환
    # (date, link)가 selected_history에 이미 있으면 no-op
    ...
    rec["selected_count"] += 1
```
- 스트라이크와 **같은 파일**(reporters.json), 같은 `{매체}::{기자}` 키.
- 멱등 추적은 스트라이크의 `history`와 분리된 `selected_history`(또는 동등
  수단)로 (date, link) 중복 방지.
- 기록 필드: 매체·기자·선택 횟수. 블랙리스트 로직과 무관(순수 관찰).

**curate.py — 대표 확정 시 호출:**
`select()`에서 selection_stats를 append하는 자리(대표 확정 직후)에서
`reporters.record_selection(rep_data, rep["source"], rep["author"], date,
rep["link"])` 호출. `rep_data`는 이미 로드되어 있고(`reporters.load()`),
non-dry-run에서 `reporters.save(rep_data)`로 영속화된다.

**리셋:** `reporters.json` 리셋(`{}`).

### D. 브리핑 개편 (daily-briefing 스킬)

`.claude/skills/daily-briefing/SKILL.md` §0 "오늘 시스템 상태" 개편:
- ❌ 당일 감점메모·매체별 감점 제거
- ✅ **전체 선택률**: 기존 순위(win/appear) 유지
- ✅ **분야별 선택률**: 분야마다 상위 매체 몇 곳(win/appear) — 매체 분야 강점
- ✅ **기자 선택 상위**: 매체·기자·선택 횟수(관찰)
- 산출물 참조 변경: `scores/articles-<date>.json` → 더 이상 없음. media.json의
  `by_category`, reporters.json의 `selected_count`를 읽는다.

## 테스트

- `update_selection_rates`: 전역 + `by_category` 동시 누적, 날짜 멱등.
- `reporters.record_selection`: 카운트 증가 + (date,link) 멱등 + 기존 레코드 호환.
- 제거 함수(`process_date` 등) 참조 테스트 정리 후 전체 스위트 그린.
- curate dry-run 선별 결과 불변(회귀 방지).

## 마이그레이션

- media.json·reporters.json 리셋 → cron 다음 실행부터 새 구조로 누적 시작.
- 선택률은 라이브 cron 누적으로만 쌓임(dry-run 미기록) → 병합·push가 관찰의 전제.
- 과거 articles-*.json 삭제는 되돌릴 필요 없는 관찰 산출물.
