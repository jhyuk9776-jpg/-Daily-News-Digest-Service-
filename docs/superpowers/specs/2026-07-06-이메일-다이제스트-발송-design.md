# 이메일 다이제스트 발송 — 설계

작성: 2026-07-06 / 이재혁 · Claude
성격: 매일 아침 생성되는 `News/YYYY-MM-DD.md` 다이제스트를 Gmail로 자동 발송하는 기능의 설계.
근거: [일정-로드맵 §7](../../../기획/일정/일정-로드맵.md) 전달 채널(이메일→카톡), M1(07-06) "이메일 발송 가동".

## 1. 목표 · 완료 조건

매일 아침 cron이 다이제스트를 생성·커밋한 직후, 그 내용을 **본인 Gmail로 발송**한다.
메일은 PC에서 읽기 좋게 HTML로 렌더링하되, **카카오톡에 그대로 복붙 가능한 plain text 블록**을 함께 제공한다.

**완료 조건:** cron 실행 시 `jhyuk9776@gmail.com`으로 오늘자 다이제스트 메일이 도착하고,
메일 하단 카톡용 블록을 복사하면 카톡(서식 없는 plain text)에서 깨지지 않고 읽힌다.

## 2. 결정 사항 (2026-07-06 확정)

| 항목 | 결정 | 근거 |
|---|---|---|

| 내용 범위 | 다이제스트 전체(모든 분야·전 건·출처 링크) | 카톡에서 바로 읽는 용도 |
| 상태 메타 | 작게 포함(생성시각·요약실패 N건) | 문제 발생 시 메일만으로 인지 |

| **결과물 제목** | "데일리 뉴스 다이제스트" → **"오늘의 뉴스 요약"** | 사용자 지정(2026-07-06). md·이메일·카톡 모두 |

제목·날짜 표기는 **`summarize.py`가 생성하는 md의 H1을 단일 출처**로 바꾸고, 이메일은 그 H1을 파싱해 그대로 물려받는다(중복 방지).

## 3. 모듈 설계

단일 책임: 다이제스트 마크다운을 메일로 조립하고 발송한다. 파싱·렌더·발송을 함수로 분리해 테스트 가능하게 한다.

### 3.0 제목·날짜 표기 (`summarize.py` 변경)

`build_markdown`의 H1을 바꾼다. 한국어 요일 포맷 헬퍼를 추가한다:
- `korean_date(date_str) -> "2026년 07월 06일 월요일"` — `["월","화","수","목","금","토","일"][weekday]` 사용
- H1: `f"# 데일리 뉴스 다이제스트 - {date}"` → `f"# 오늘의 뉴스 요약 - {korean_date(date)}"`
- `notify.py`는 이 H1 텍스트를 파싱해 제목·날짜를 그대로 물려받는다(표기 단일 출처).

### 3.1 파싱 — `parse_digest(md_text) -> Digest`

입력 마크다운 구조(변경 후 `summarize.py` 출력):
```
# 오늘의 뉴스 요약 - 2026년 07월 06일 월요일
> 생성: 2026-07-06 07:55 KST · 요약실패 1건 · 호출오류 0건 · 추출실패 0건
## 경제
### 기사 제목
- 불릿1
- 불릿2
- 출처: [한국경제](https://...) 외 관련 1건
## 사회
...
```

산출 구조(예: dict/dataclass):
- `title_line`: H1의 `# ` 이후 텍스트 전체(`오늘의 뉴스 요약 - 2026년 07월 06일 월요일`) — subject·본문 제목에 그대로 사용
- `meta`: 인용줄 원문 문자열(`생성: … · 요약실패 1건 …`)
- `categories`: `[{name: "경제", articles: [{title, bullets:[...], source:{label, url, extra}}]}]`
  - 출처줄은 불릿과 구분: `- 출처: [label](url) 외 관련 N건` 패턴을 정규식으로 분리(`label`,`url`,`extra="외 관련 1건"`)
  - 일반 불릿은 `bullets`에 텍스트로 축적
- 기사 수 합계(`total_count`)는 subject에 사용

### 3.2 렌더 — `render_email(digest) -> (subject, html_body, text_body)`

- **subject:** `📰 {title_line} ({total_count}건)` → 예: `📰 오늘의 뉴스 요약 - 2026년 07월 06일 월요일 (16건)`
- **text_body (카톡용 plain):**
  - 맨 위 제목줄: `{title_line}`(= `오늘의 뉴스 요약 - 2026년 07월 06일 월요일`)
  - 분야 구분선: `──────── 경제 ────────`
  - 기사 제목: `▪ {title}`
  - 불릿: `  · {bullet}`
  - 출처: `  출처: {label} {url}` (마크다운 링크 문법 없이 URL 평문)
  - 하단 메타: `{meta}` 한 줄
  - 마크다운 문법(`[]()`, `#`, `>`)이 남지 않아야 함
- **html_body:**
  - 분야 `<h2>`, 기사 `<h3>`, 불릿 `<ul><li>`, 출처는 `<a href>` 링크
  - 본문 하단에 **카톡용 블록** 섹션: `<h3>─ 카카오톡용 (아래 블록만 복사) ─</h3>` + `text_body`를 `<pre>`로 감싸 monospace·복붙 친화
  - 메타는 작은 회색 글씨(`<p style="color:#888;font-size:12px">`)
  - 인라인 스타일만 사용(메일 클라이언트 CSS 제약)

### 3.3 발송 — `send_email(subject, html_body, text_body, cfg)`

- `MIMEMultipart("alternative")` — part 순서 plain → html(클라이언트는 html 우선 표시, plain은 폴백)
- `smtplib.SMTP_SSL("smtp.gmail.com", 465)` → `login(cfg.address, cfg.app_password)` → `send_message`
- 발신 `From = cfg.address`, 수신 `To = cfg.to`(기본 `cfg.address`)
- 헤더 `Subject`(UTF-8 인코딩), `From`, `To`

### 3.4 진입점 — `main()`

1. `date = today(KST)`; `path = News/<date>.md`
2. 파일 없으면 `"발송 스킵: 오늘 다이제스트 없음"` 출력 후 exit 0
3. 환경변수 로드: `GMAIL_ADDRESS`, `GMAIL_APP_PASSWORD`(필수), `NOTIFY_TO`(선택). 필수 누락 시 명확한 에러로 exit 1
4. `parse → render → send`. 성공 시 `"발송 완료: {to}"` 출력
5. 발송 예외는 스택 대신 원인 요약을 출력하고 exit 1(스텝 빨간불)

## 4. 설정 — 환경변수 = GitHub Secrets

| 이름 | 용도 | 필수 |
|---|---|---|
| `GMAIL_ADDRESS` | 발신 Gmail 주소(기본 수신자) | ✅ |
| `GMAIL_APP_PASSWORD` | Gmail **앱 비밀번호**(로그인 비밀번호 아님, 16자리) | ✅ |
| `NOTIFY_TO` | 수신자 override(미설정 시 `GMAIL_ADDRESS`) | ⬜ |

- 앱 비밀번호는 2단계 인증 활성 계정에서 발급. 유출 시 해당 키만 폐기하면 됨.

## 5. 워크플로 변경 — `.github/workflows/daily.yml`

- 기존 "다이제스트·점수 자동 커밋" 스텝 **뒤에** "이메일 발송" 스텝 추가:
  ```yaml
  - name: 이메일 발송
    env:
      GMAIL_ADDRESS: ${{ secrets.GMAIL_ADDRESS }}
      GMAIL_APP_PASSWORD: ${{ secrets.GMAIL_APP_PASSWORD }}
    run: python3 src/notify.py
  ```
- 커밋이 앞서므로 발송 실패가 다이제스트 저장을 막지 않음. 발송 실패 시 스텝이 빨간불.

## 6. 테스트 — `tests/test_notify.py` (unittest)

- `korean_date`(summarize): `2026-07-06` → `2026년 07월 06일 월요일`, 요일 매핑 정확성(월=0)
- `parse_digest`: 샘플 md → `title_line`("오늘의 뉴스 요약 - …") 추출, 분야 수·기사 수·출처 label/url/extra 분리 정확성, 메타 추출
- `render_email`:
  - subject에 `오늘의 뉴스 요약`·요일 날짜·건수 포함
  - html에 분야 heading·출처 `<a href>`·카톡용 `<pre>` 블록 존재
  - text_body가 `▪` 사용, 마크다운 링크(`](`) 미포함, URL 평문 포함, 메타 한 줄 포함
- `send_email`: `smtplib.SMTP_SSL`을 mock → `login(address, app_password)`·`send_message` 호출과 multipart(2 part) 검증
- `main`: 오늘 파일 없을 때 발송 스킵(exit 0), 필수 env 누락 시 exit 1

## 7. 범위 밖 (YAGNI)

- 로컬 발송 UX·CLI 플래그, 다중 수신자·구독 관리, 발송 실패 별도 JSON 로깅, HTML 템플릿 엔진.
- 카톡 직접 발송(API) — 로드맵 §7 C안, 후순위.
