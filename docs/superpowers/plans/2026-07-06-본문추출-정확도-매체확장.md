**Goal:** 본문 추출이 "추천기사 위젯"이 아니라 진짜 기사 본문(수치 포함)만 잡도록 고치고(ISSUE-001·002 해결), 선별 견고성을 위해 매체 7곳을 추가한다.

**Architecture:** `extract.py`를 (1) 순수 파싱 `_parse_body(html,url,title)` + (2) 네트워크 래퍼 `extract_body(url,title)`로 분리해 저장 HTML 픽스처로 테스트한다. 도메인별 본문 선택자 맵 + 검증 가드 `looks_like_body`로 정확도를 확보하고, `iter_contents`를 본문 우선 순서로 바꿔 수치를 살린다. 요약 프롬프트에 수치 보존 규칙을 넣고, `sources.yaml`에 신규 매체를 검증 후 추가한다.

**Tech Stack:** Python(표준 라이브러리 + 기존 의존성 `requests`·`beautifulsoup4`·`feedparser`만). 테스트는 표준 `unittest` + 저장 HTML 픽스처. 실행 `python3 -m unittest discover -s tests`.

## Global Constraints

- 새 pip 의존성 추가 금지(`requirements.txt` 불변). 기존 `requests`·`bs4`·`feedparser`만 사용.
- 테스트는 네트워크를 타지 않는다(`_parse_body`는 저장 HTML 픽스처로, `extract_body`는 `requests.get` mock으로).

- `extract.py`는 이미 `from __future__ import annotations`가 있어 `str | None` 표기 사용 가능(로컬 py3.9/CI py3.11 공용).
- 테스트 파일은 `sys.path.insert(0, os.path.join(os.path.dirname(__file__), "..", "src"))` 후 모듈 import(기존 관례).

```python
_DATE_RE = re.compile(r"20\d{2}[.\-/]\s?\d{1,2}[.\-/]\s?\d{1,2}")
_TITLE_STOPWORDS = {"오늘", "관련", "기자", "뉴스", "속보", "단독"}


def _title_keywords(title: str) -> list[str]:
    toks = re.findall(r"[가-힣A-Za-z0-9]{2,}", title)
    return [t for t in toks if t not in _TITLE_STOPWORDS]


def looks_like_body(text: str, title: str = "") -> bool:
    """추출 텍스트가 진짜 기사 본문인지 검증한다(추천위젯·제목불일치 기각)."""
    if len(text) < MIN_BODY:
        return False
    if len(_DATE_RE.findall(text)) >= 3:
        return False
    keywords = _title_keywords(title)
    if keywords and not any(k in text for k in keywords):
        return False
    return True
```

```python
# 도메인 → 진짜 본문 컨테이너 CSS 선택자 후보(있으면 우선, 없으면 휴리스틱 폴백).
SITE_SELECTORS: dict[str, list[str]] = {
    "hankyung.com": ["#articletxt", ".article-body", "[itemprop='articleBody']"],
    # 나머지 도메인은 Task 6에서 확인해 채운다(폴백+가드가 안전망).
}


def _domain(url: str) -> str:
    host = urlparse(url).netloc.lower()
    return host[4:] if host.startswith("www.") else host


def _parse_body(html: str, url: str, title: str = "") -> str | None:
    soup = BeautifulSoup(html, "html.parser")
    for tag in soup(["script", "style", "nav", "header", "footer", "aside"]):
        tag.decompose()
    container = None
    for sel in SITE_SELECTORS.get(_domain(url), []):
        container = soup.select_one(sel)
        if container:
            break
    scope = container or soup.find("article") or soup
    paragraphs = scope.find_all("p")
    text = " ".join(p.get_text(" ", strip=True) for p in paragraphs)
    text = " ".join(text.split())[:MAX_CHARS]
    if not looks_like_body(text, title):
        return None
    return text
```

```python
def iter_contents(item: dict):
    """요약에 쓸 텍스트 후보를 우선순위 순으로 하나씩 내보낸다.

    각 후보 매체마다 검증 통과한 '본문'을 먼저, 이어서 충분한 RSS 요약을 내보낸다.
    본문 우선이라 원문의 수치가 살아난다(ISSUE-002). 본문은 제목으로 가드된다(ISSUE-001).
    """
    title = item.get("title", "")
    for source, link, summary in candidate_sources(item):
        body = extract_body(link, title)
        if body:
            yield {"text": body, "content_source": source, "method": "body", "link": link}
        if summary and len(summary) >= MIN_SUMMARY:
            yield {"text": summary, "content_source": source, "method": "rss", "link": link}
```
