# 웹 다이제스트 JSON 계약 + 최소 조회화면 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 파이프라인이 다이제스트를 구조화 JSON으로 내보내고, 그 JSON을 읽어 분야별로 보여주는 최소 Next.js 조회화면을 만든다.

**Architecture:** `summarize.py`에 `collect_digest(selected, results)`를 추가해 마크다운과 JSON이 같은 구조에서 파생되게 한다. 파이프라인은 `web/public/data/<date>.json` + `latest.json`을 쓴다. `web/`의 Next.js(App Router, 서버 컴포넌트)가 빌드 시 `latest.json`을 읽어 `Digest` 컴포넌트로 렌더한다.

**Tech Stack:** Python 3.9 (stdlib unittest) · Next.js 15 (App Router) · React 19 · TypeScript(strict) · vitest + @testing-library/react

## Global Constraints

- 파이썬 테스트는 `python3 -m unittest discover -s tests`로 돌고, 네트워크/API를 쓰지 않는다.
- `build_markdown(selected, results, counters)` **시그니처는 바꾸지 않는다**(tests/test_fallback.py, tests/test_notify.py가 직접 호출). 내부만 `collect_digest`로 리팩터.
- JSON은 `json.dumps(..., ensure_ascii=False, indent=2)`로 쓴다(한글 그대로).
- 분야 순서는 `selected["categories"]`의 순서를 그대로 보존한다(경제→사회→세계→IT/테크).
- `related_links`는 `{"source","link"}` 두 필드로만 정규화한다(원본의 extra 필드 버림).
- 웹은 TypeScript `strict: true`. import는 상대경로(`../lib/types`) 사용 — path alias 미설정(vitest 설정 단순화).
- 웹 산출물(`node_modules/`, `.next/`)과 생성된 `public/data/*.json`은 git에서 무시하되, 개발용 `latest.json` 샘플만 강제 추가한다.
- 새 의존성은 이 문서에 명시된 것만. 임의 추가 금지.

**입력 데이터 구조 (참고):**
- `selected = {"date": "YYYY-MM-DD", "categories": {"경제": [item, ...], "사회": [...], ...}}`
- `item = {"title","link","source","related_links":[{"source","link"},...], ...(그 외 필드)}`
- `results = {item["link"]: (bullets:list[str], status:str)}` — 요약 성공 항목만 키로 존재.

```python
def collect_digest(selected: dict, results: dict) -> dict:
    """선별 항목+요약 결과를 웹 계약(JSON) 구조로 모은다.

    마크다운 렌더와 JSON 직렬화가 이 단일 구조에서 파생돼 서로 어긋나지 않는다.
    요약 성공(link in results)한 항목만 담고, 분야는 빈 분야도 포함한다.
    """
    categories = []
    for category, items in selected["categories"].items():
        ok = []
        for item in items:
            if item["link"] not in results:
                continue
            bullets = results[item["link"]][0]
            ok.append({
                "title": item["title"],
                "bullets": bullets,
                "source": item["source"],
                "link": item["link"],
                "related_links": [
                    {"source": rl["source"], "link": rl["link"]}
                    for rl in item.get("related_links", [])
                ],
            })
        categories.append({"name": category, "items": ok})
    return {"date": selected["date"], "categories": categories}
```
