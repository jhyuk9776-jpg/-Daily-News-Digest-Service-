# 포트폴리오(산출물 1) 집필 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 채용(기획·PM 신입) 대상 포트폴리오 한 편을 마크다운으로 완성한다 — "기획력 6장면" 서사.

**Architecture:** 단일 마크다운 파일(`docs/portfolio/포트폴리오-기획력.md`)을 헤드라인+도입 → 6장면 → 마무리 순으로 한 섹션씩 집필·커밋한다. 각 장면은 `①질문 ②결정 ③트레이드오프 ④증거` 미니 틀을 공유한다. 프로세스이므로 코드 테스트 대신 **증거 앵커 실재 검증**(파일·§·커밋·라이브 링크가 실제로 존재하는지)이 각 task의 통과 조건이다.

**Tech Stack:** Markdown → PDF 렌더. 검증은 `grep`/`Read`/`git log`/`ls`로 앵커 실재를 확인.

**메인 형태: PDF** (2026-07-10 확정). 소스는 마크다운 한 벌, 거기서 PDF로 렌더. 제품
앱(`web/`)에는 넣지 않는다.

## Global Constraints

- 대상: 채용 담당자·면접관 / 지원 포지션: 기획·PM **신입**(PO 제외).

- 톤: 꼼꼼하되 적당히 자신감. 모든 주장에 **③트레이드오프(포기)**와 **④증거**를 붙인다.
- 정직성: 실제 코드·문서·커밋·라이브에 존재하는 것만 근거로. **지어낸 성과 지표 금지.** 상용화(장면 5)는 "달성"이 아니라 "기획(구조·원칙)"으로만, 미정 숫자는 미정으로 노출.
