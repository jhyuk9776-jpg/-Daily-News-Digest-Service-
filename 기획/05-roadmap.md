# 로드맵

## 1. Phase 1 목표

사실 중심 데일리 뉴스 다이제스트를 매일 생성하는 최소 시스템을 만든다.

## 2. 1주차 계획

| 단계 | 목표 | 완료 조건 |
|---|---|---|
| Day 1 | 기획 확정 | 분야, 출처, 기사 수, 저장 형식 결정 |
| Day 2 | 수집 기능 구현 | 고정 소스에서 기사 리스트 수집 |
| Day 3 | 선별/중복 제거 구현 | 분야별 후보 기사 정리 |
| Day 4 | AI 요약/마크다운 생성 | 날짜별 다이제스트 파일 생성 |
| Day 5 | 테스트/자동화 | 실제 실행 결과 검수 및 자동 실행 연결 |

## 2.1 오늘 상세 계획

| 날짜 | 목표 | 작업 |
|---|---|---|

| 2026-06-29 | 비용 기준 정리 | AI 요약 비용 가정과 첫 달 예산 설정 |

## 3. Phase 1 산출물

- 기획 문서
- 뉴스 출처 설정 파일
- Python 수집 스크립트
- AI 요약 프롬프트
- 날짜별 다이제스트 마크다운
- 실행 방법 문서
- 자동화 설정

## 4. Phase 2 후보

| 후보 | 설명 |
|---|---|
| Supabase 저장 | 웹/앱에서 조회할 수 있도록 DB에 저장 |
| 웹/PWA | 모바일에서도 보기 좋은 간단한 화면 제공 |
| 이메일 발송 | 매일 아침 메일로 자동 전달 |
| 관심 분야 설정 | 사용자가 보고 싶은 분야만 선택 |
| 불편/만족 신호 분석 | 뉴스에서 사업 아이디어의 단서를 뽑는 기능 |

## 5. 우선순위 원칙

1. 먼저 매일 쓸 수 있는 결과물을 만든다.
2. 자동화보다 결과물 품질을 먼저 검증한다.
3. 웹/앱보다 데이터 파이프라인을 먼저 안정화한다.
4. 판단과 해석은 Phase 1에서 의도적으로 줄인다.
