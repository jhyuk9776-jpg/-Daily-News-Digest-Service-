# 로드맵

## 1. Phase 1 목표

사실 중심 데일리 뉴스 다이제스트를 매일 생성하는 최소 시스템을 만든다.

## 2. 1주차 계획

| 단계 | 목표 | 완료 조건 |
|---|---|---|
| Day 1 | 기획 확정 | 분야, 출처, 기사 수, 저장 형식 결정 |
| Day 2 | 수집 기능 구현 | 고정 소스에서 기사 리스트 수집 |
| Day 3 | 선별/중복 제거 구현 | 분야별 후보 기사 정리 |
| Day 4 | AI 요약/마크다운 생성 | 날짜별 다이제스트 파일 생성 |
| Day 5 | 테스트/자동화 | 실제 실행 결과 검수 및 자동 실행 연결 |

## 5. 우선순위 원칙

1. 먼저 매일 쓸 수 있는 결과물을 만든다.
2. 자동화보다 결과물 품질을 먼저 검증한다.
3. 웹/앱보다 데이터 파이프라인을 먼저 안정화한다.
4. 판단과 해석은 Phase 1에서 의도적으로 줄인다.

## 6. Phase 2 후보 상세: 매체 객관성 동적 점수

로드맵 4절의 "출처 신뢰도 티어"를 **동적 객관성 점수**로 확장한 아이디어(2026-07-01).
Phase 1의 "객관적 서술 우선" 원칙(결정 로그 1.3)에 숫자 기반 학습을 붙여, 매체
신뢰도가 피드백으로 수렴하게 한다. **지금 구현하지 않는다** — 임계값·증감폭을
정하려면 피드백 데이터가 먼저 쌓여야 하며, 근거 없이 숫자를 정하면 추측이 된다.

### 초안 아이디어(사용자 제안)

- 모든 매체 100점에서 시작한다.
- 객관적이라 피드백받은 매체는 가점(예: +1), 감정적/평가 표현이 나오면 감점.
- 일정 점수(예: 80점) 이하로 내려가면 제외 후보로 두고 새 매체를 찾는다.
- 좋게 평가된 매체는 점수가 올라 신뢰도가 누적된다.
- 감점 트리거 예: "~가 다 했네"처럼 평가어가 제목/본문에 포함되면 감점
  (감점 사전은 AI_CONTEXT §6 "피해야 할 표현"에서 시드).

### 구현 전 정할 점(다듬은 방향)

| 쟁점 | 문제 | 방향 |
|---|---|---|
| 점수 단위 | 좋은 매체도 특정 기사만 감정적일 수 있어, 기사 하나로 매체 전체를 깎으면 거칠다 | 기사별 객관성 점수로 선별하고, 그 결과를 매체 점수에 천천히 반영(느린 평판) |
| 누적 vs 비율 | 기사를 많이 내는 매체는 양 때문에 감점이 더 쌓임 | 누적합 대신 기사당 평균/이동평균으로 정규화 |
| 임계값·증감폭 | 80점·±1은 임의값. 회복·하락 속도가 결과를 좌우 | 초기엔 제외 없이 기록만 하다가 점수 분포를 보고 임계값 결정 |
| 새 매체 자동 탐색 | RSS 자동 발굴은 별개의 큰 기능 | 당분간 수동 교체(제외 후보만 사용자에게 알림) |
| 키워드 감점 오탐 | "충격" 등은 사실 기사(지진 충격 등)에도 등장 | 단순 매칭은 거칠므로 초기엔 보조 신호로만 사용 |

### 연료(선행 작업)

이 시스템은 라벨 데이터가 연료다. Phase 1 동안 "객관적 예시 수집"(결정 로그 1.3의
네이버 링크 1호처럼 사용자가 좋다고 고른 기사)을 꾸준히 모아두면, Phase 2에서 임계값과
감점 사전을 실제 데이터로 튜닝할 수 있다. 실제 구현 시 별도 브레인스토밍을 거친다.
