# 시스템 기획

> **어떻게 만드는가**를 다루는 폴더. (로드맵이 예고했던 `구현기획`을 이 이름으로 확정.)
> **왜·누구를 위해·어떻게 파는가**는 [../서비스기획/](../서비스기획/)를 본다.

이 폴더는 데일리 뉴스 다이제스트의 **시스템 설계**를 한곳에 모은다. 전체 아키텍처와
기능별 상세 설계(어떤 방법으로 구현하는지)가 여기 있고, 에이전트 실행용 구현 계획
체크리스트는 관례 위치(`docs/superpowers/plans/`)에 그대로 두고 링크로 연결한다.

## 경계 (서비스기획 vs 시스템기획)

- **서비스기획** = 왜/누구/판매(요금제·타깃·수익). 숫자·전략.
- **시스템기획** = 어떻게 만드는가(아키텍처·기능 설계·구현 방식). 부품·규칙.
- 둘이 겹치는 로드맵·결정 로그는 상위 [../](../)의 `05-roadmap.md`·`04-decision-log.md` 참조.
