# 작업일지

데일리 뉴스 다이제스트 프로젝트의 날짜별 작업 기록.
**목적:** 서비스 완성 후 하나의 기획서로 엮기 위한 재료. 그래서 "무엇을 했는지"뿐 아니라
**"왜 그렇게 결정했는지(고민한 배경과 트레이드오프)"**를 함께 남긴다.
각 일자를 **원래 일정(계획)** 과 **실제 작업(한 것)** 으로 나눠 기록한다.

### 실제 작업

- 구현은 시작하지 않고 **범위를 선명하게 고정하는 데 집중**.

### 주요 결정과 그 이유 (고민한 지점)

- **"판단 없는 사실 중심" 다이제스트로 방향을 잡음**
  - 고민: 뉴스 서비스는 요약·논평·전망까지 하기 쉬운데, 그러면 AI의 판단이 개입된다.
  - 결정 근거: 타깃 사용자는 "뉴스를 많이 읽고 싶은 사람"이 아니라 "세상의 흐름을
    효율적으로 파악하려는 사람". 그래서 **요약은 판단을 대신 내리지 않고 판단할 재료를
    정리**하는 것으로 원칙을 세움. 평가어("논란이 커지고 있다" 등)를 의도적으로 배제.

- **정치 분야 Phase 1 제외**
  - 고민: 정치는 트래픽이 크지만 논쟁·판단을 유발한다.
  - 결정 근거: 사실 중심 원칙과 충돌하고 초기 범위를 키운다. 제외하되 추후 옵션 처리 여지만 남김.

- **하루 8건(분야별 2건), 저장은 `News/YYYY-MM-DD.md`**
  - 고민: 몇 건이 "충분하되 부담 없는" 양인가.
  - 결정 근거: **읽는 시간 5분** 기준. 많이 주기보다 매일 끝까지 읽게 만드는 게 우선.
    결과물은 웹/DB보다 먼저 **파일 하나로 매일 쓸 수 있게**(작게 시작 원칙).

- **신뢰성 > 요약 품질**
  - 결정 근거: 요약이 매끄러운 것보다 **출처·날짜·중복 제거의 신뢰성**이 먼저다.
    그래서 "출처 링크는 반드시 남긴다"를 하드 원칙으로 둠.

- **로컬 Python + 무료 RSS, 자동화는 GitHub Actions 후보**
  - 고민: 비용을 어디에 쓸 것인가.
  - 결정 근거: 무료 RSS·로컬 실행으로 고정비를 0에 가깝게 두고 **비용은 AI 요약 API에만
    집중**. 자동화는 무료이고 전원/PC와 무관한 GitHub Actions를 우선 검토.

- **웹/앱 스택 TypeScript(Next.js + Expo) 확정 — 단, Phase 3로 미룸**
  - 결정 근거: 언어 하나로 웹+iOS+안드로이드를 커버해 **솔로 개발 부담이 가장 작다**.
    Supabase JS SDK 지원이 두터워 Phase 2 DB와 직결.
  - 미룬 이유: **데이터 파이프라인을 먼저 안정화**한다는 우선순위 원칙. 지금은 기록만.

- **비용: 첫 달 22달러 (Claude Code 월 구독 기준)** — 비용 늘어나는 선택은 먼저 가정을 밝힌다는 규칙.

## 2일차 — 2026-06-30 (로드맵 Day 2·3·4 압축: 수집 → 선별 → 요약)

### 원래 일정 (로드맵 기준 3일 분량)
- Day 2: 수집 기능 구현 — 고정 소스에서 기사 리스트 수집
- Day 3: 선별/중복 제거 구현 — 분야별 후보 기사 정리
- Day 4: AI 요약/마크다운 생성 — 날짜별 다이제스트 파일 생성

### 실제 작업 (하루에 3단계 완주 → 파이프라인 end-to-end 완성)
1. **수집(`src/fetch.py`)** — `sources.yaml`의 분야별 RSS를 `feedparser`로 수집,
   발행시각 ISO 정규화, `raw/YYYY-MM-DD.json` 저장. 한 출처 실패가 전체를 막지 않게 예외 격리.
2. **선별/중복제거(`src/select.py` → 이후 `curate.py`)** — 날짜 필터, 증거점수 휴리스틱,
   제목 유사도 클러스터링(같은 사건 묶기)+교차검증 매체 수 집계, 분야별 상위 2건.
3. **AI 요약/마크다운(`src/extract.py`, `src/summarize.py`)** — 본문 확보 후 Claude Haiku
   4.5로 사실 1~3문장 요약(기사별 개별 호출)+URL 캐시, 출처 1:1 부착, `News/2026-06-30.md` 생성.
- 각 단계마다 설계 메모를 먼저 남기고 구현: `docs/superpowers/specs/2026-06-30-day2~4-*.md`

### 주요 결정과 그 이유 (고민한 지점)

- **수집 단계에서 하드 필터를 하지 않고 "넓게 수집 → 선별에서 랭킹"으로 감** (증거 기반 품질 전략)
  - 고민: 처음엔 "증거(숫자·통계) 있는 기사만 남기자"는 하드 필터를 생각했다.
  - 문제 발견: 증거는 통계만이 아니라 **판결·정책 발표·공식 발언**도 포함한다. 하드 필터를
    걸면 중요한 뉴스를 버리고 **분야별 2건을 못 채울 위험**이 있다.
  - 결정: 수집은 넓게 하고, 증거·교차검증을 **선별 단계의 순위 신호(가점)**로만 쓴다.

- **교차검증을 신뢰 신호 + 중복 제거에 함께 사용**
  - 결정 근거: 2개 이상 독립 매체가 같은 사실을 보도하면 **신뢰도가 올라가고**, 동시에
    같은 사건을 하나로 묶어 **중복도 줄인다**. 한 로직으로 두 목적을 달성.

- **요약 모델로 Claude Haiku 4.5 채택**
  - 고민: 1일차엔 "OpenAI 저비용 모델 우선"이었다.
  - 결정 근거: 하루 8건·고정 형식 요약이라 **저비용 소형 모델로 충분**(월 ~$1 예상).
    품질/비용 균형에서 Haiku를 택함.

- **API를 Anthropic 콘솔이 아닌 Replicate 경유로 호출** (실무에서 막혀 우회한 지점)
  - 고민: Anthropic 콘솔 결제가 **카드/본인 검증 문제로 막혔다**.
  - 결정: 같은 모델(`anthropic/claude-4.5-haiku`)을 **Replicate 공식 채널**로 사용.
    제약(최소 max_tokens, 잔액 $5 미만 시 분당 6건 제한)은 429 시 대기·재시도로 흡수.

- **본문 추출은 "빈/짧은 요약만" 보강, 요약은 기사별 개별 호출**
  - 결정 근거: 대상이 **최종 8건뿐**이라 본문 추출·개별 호출의 비용이 거의 0.
    개별 호출은 **격리성·출처 1:1 정확성·URL 캐시** 이점이 커서 선택.

- **"출처 없는 사실은 싣지 않는다"를 요약 단계 원칙으로 못박음** — 신뢰성 우선(1일차 원칙)의 구현.

### 주요 결정과 그 이유 (고민한 지점)

- **선별 정렬에서 증거점수를 뺌** (기준을 바꾼 판단)
  - 문제 발견: 증거점수가 최대 5점에 몰려 변별력이 약하고, **감정적 제목도 숫자만 있으면 고득점**.
  - 결정: 랭킹에서 증거점수를 빼고 교차검증수→매체우선순위→최신순으로. 증거점수는 기록용
    메타데이터로만 남김. 객관성 반영은 데이터를 모아 Phase 2에서 제대로 만든다.

- **요약 폴백을 체인으로 설계** (조용한 실패를 막는 방어선)
  - 문제 발견: 짧은 RSS(102자)가 길이 기준을 통과해 부실 요약 → 모델이 대화체 응답 →
    불릿 0개 → 비결정적 실패(매일경제 기사).
  - 결정: 프롬프트를 강화("불릿만, 낼 사실 없으면 아무것도 출력 안 함")하고, 불릿 0개면
    전체 본문 재요약 → 같은 사건 다음 우선순위 매체 본문으로 순차 폴백. `tests/test_fallback.py`로 고정.

- **자동화는 "초록불"이 아니라 "결과물"로 검증해야 한다** (오늘의 핵심 교훈)
  - 고민: 워크플로우가 성공(초록불)으로 떠서 끝난 줄 알았으나, 생성물을 열어보니
    **8건 요약이 전량 실패**하고 링크-only로 폴백돼 있었다. 원인은 Replicate **토큰 401**.
  - 뿌리 원인: 링크-only 폴백은 *개별 기사* 최후 수단인데, *전량 인증 실패*까지 성공으로
    넘겨 **부실 다이제스트가 그대로 main에 자동커밋**됐다. 초록불이 품질 실패를 가림.
  - 결정: `summarize.run()`에 **요약 과반 실패(FAIL_RATIO 0.5) 시 exit 1** 가드 추가.
    파일은 남기되(아티팩트 확인용) `run.sh`의 `set -e`가 자동커밋 단계를 막아 오염 차단.
    테스트 4건 추가. (커밋 `03b7ff1`)
  - 검증: 토큰 무효 상태에서 실제로 CI 빨간불·커밋 차단됨을 확인 → 토큰 갱신 후 재실행에서
    **요약 8건·실패 0**으로 통과·자동커밋까지 확인. (가드가 없었다면 오늘도 쓰레기가 커밋될 뻔)

- **다이제스트 검수에서 "감점 예시 1호" 확보** (Phase 2 연료)

### Phase 2 첫 기능 설계 (객관성 스코어러)

Phase 1을 마치고, Phase 2 첫 기능을 브레인스토밍→설계→구현계획까지 확정(코딩은 다음 세션).

- **아이디어(사용자 제안):** 요약한 8건뿐 아니라 **수집·필터링한 기사 전량**의 점수를 매겨

- **주요 결정과 이유:**
